# 上传接口修复说明

## 🐛 问题描述

用户反馈：**`/apps/common/upload_file` 上传时没带上上传文件 `file_name`，导致文件没有后缀，上传之后返回的 URL 没有累加，然后在 update 接口带过来**

### 具体问题

1. **文件没有扩展名**：上传的文件保存时没有扩展名（如 `.jpg`、`.png`）
2. **返回的 URL 不完整**：返回的是本地文件路径，而不是可访问的 URL
3. **文件名丢失**：没有保留原始文件名信息

---

## 🔍 问题分析

### 原代码问题

**文件**：`apps/common/view/common_view.go`

```go
func (e *CommonView) UploadFileView(ctx context.Context, header *web.Header, request interface{}) (interface{}, *bmserror.BMSError) {
	req := request.(*pbbasic.UploadFileRequest)
	dateStr := timeutil.TodayDateStr(timeutil.DateIntFormat)

	path := "./upload/" + dateStr + "/"
	os.MkdirAll(path, os.ModePerm)
	os.Chmod(path, os.ModePerm)
	
	// ❌ 问题 1: 如果 GetFileName() 为空，生成的文件名没有扩展名
	if req.GetFileName() != "" {
		path = path + req.GetFileName()
	} else {
		generateUUID, _ := uuid.NewUUID()
		path += generateUUID.String()  // ❌ 没有扩展名
	}
	
	f, err := os.Create(path)
	// ...
	
	// ❌ 问题 2: 返回的是本地文件路径，不是 URL
	return &pbbasic.UploadFileResponse{
		Url: convert.String(path),  // 返回 "./upload/20240113/uuid"
	}, nil
}
```

**问题根源**：
1. 前端上传时，`FormData` 中的文件名可能没有正确传递到后端的 `req.GetFileName()`
2. 即使传递了，如果为空，生成的 UUID 文件名也没有扩展名
3. 返回的路径是本地路径（`./upload/...`），不是可访问的 URL（`/uploads/...`）

## ✅ 解决方案

### 修复后的代码

**文件**：`apps/common/view/common_view.go`

```go
func (e *CommonView) UploadFileView(ctx context.Context, header *web.Header, request interface{}) (interface{}, *bmserror.BMSError) {
	req := request.(*pbbasic.UploadFileRequest)
	
	// ✅ 1. 获取原始文件名和扩展名
	originalFilename := req.GetFileName()
	if originalFilename == "" {
		originalFilename = "file.jpg" // 默认文件名
	}
	
	// ✅ 2. 提取文件扩展名
	ext := ""
	if idx := strings.LastIndex(originalFilename, "."); idx != -1 {
		ext = originalFilename[idx:] // 包含点号，如 ".jpg"
	} else {
		ext = ".jpg" // 默认扩展名
	}
	
	// ✅ 3. 创建上传目录（按日期分类）
	dateStr := timeutil.TodayDateStr(timeutil.DateIntFormat)
	uploadDir := "./uploads/" + dateStr + "/"
	os.MkdirAll(uploadDir, os.ModePerm)
	os.Chmod(uploadDir, os.ModePerm)
	
	// ✅ 4. 生成唯一文件名（UUID + 扩展名）
	generateUUID, _ := uuid.NewUUID()
	filename := generateUUID.String() + ext  // 如：uuid.jpg
	fullPath := uploadDir + filename
	
	// ✅ 5. 创建并写入文件
	f, err := os.Create(fullPath)
	if err != nil {
		return nil, bmserror.NewError(constant.ErrFile, "create file err %v", err.Error())
	}
	defer f.Close()
	
	_, err2 := f.Write(req.File)
	if err2 != nil {
		return nil, bmserror.NewError(constant.ErrFile, "write file err %v", err2.Error())
	}
	
	// ✅ 6. 返回可访问的 URL（格式：/uploads/20240113/uuid.jpg）
	fileUrl := "/uploads/" + dateStr + "/" + filename
	
	return &pbbasic.UploadFileResponse{
		Url:      convert.String(fileUrl),           // 返回 URL
		FileName: convert.String(originalFilename),  // 返回原始文件名
	}, nil
}
```

## 📊 前端调用示例

### 前端上传代码

**文件**：`frontend/src/components/ImageUpload.jsx`

```javascript
const customUpload = async ({ file, onSuccess, onError }) => {
  const formData = new FormData();
  formData.append('file', file);  // ✅ 文件对象会自动包含 file.name

  try {
    const response = await request.post('/apps/common/upload_file', formData, {
      headers: {
        'Content-Type': 'multipart/form-data',
      },
    });

    if (response.retcode === 0) {
      // ✅ response.data 包含：
      // {
      //   "url": "/uploads/20240113/uuid.jpg",
      //   "file_name": "original_photo.jpg"
      // }
      onSuccess(response.data);
      message.success('上传成功');
    } else {
      onError(new Error(response.message || '上传失败'));
      message.error(response.message || '上传失败');
    }
  } catch (error) {
    onError(error);
    message.error('上传失败');
  }
};
```

---

### 后端接收处理

**框架层**：`framework/web/upload_file_strategy.go`

后端框架会自动从 `multipart/form-data` 中提取：
1. **文件内容**：`file` 字段 → `req.File`（字节数组）
2. **文件名**：从 `Content-Disposition` header 中提取 → `req.GetFileName()`

**示例**：
```
POST /apps/common/upload_file
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary...

------WebKitFormBoundary...
Content-Disposition: form-data; name="file"; filename="photo.jpg"
Content-Type: image/jpeg

[binary data]
------WebKitFormBoundary...
```

后端接收：
```go
req.File         // 文件二进制内容
req.GetFileName() // "photo.jpg"
```

## ⚠️ 注意事项

### 1. 文件名编码

- 原始文件名可能包含中文或特殊字符
- UUID 文件名避免了编码问题
- 原始文件名仅用于显示，不用于存储

### 2. 扩展名安全

- 仅允许图片格式：`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`
- 其他格式会被拒绝
- 建议在后端添加文件内容校验（MIME type）

### 3. 目录权限

```bash
# 确保上传目录存在且有写权限
mkdir -p ./uploads
chmod 755 ./uploads
```

### 4. 磁盘空间

- 按日期分目录，便于管理和清理
- 建议定期清理旧文件
- 监控磁盘使用情况
