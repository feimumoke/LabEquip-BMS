## 🔐 认证流程

### 完整的请求流程

```
1. 前端发送请求
   ├─ Header: Authorization: Bearer <token>
   └─ Header: X-User-Email: user@example.com

2. 后端 parseRequestHeader 函数
   ├─ 检查是否是白名单路径
   │  ├─ 是 → 跳过验证，直接返回
   │  └─ 否 → 继续验证
   │
   ├─ 从 HTTP Header 提取 token
   │  ├─ Authorization: Bearer <token>
   │  └─ 或 Token: <token>
   │
   ├─ 调用 AuthService.ValidateToken()
   │  ├─ 查询 login_session_tab 表
   │  ├─ 检查 session 是否过期
   │  └─ 解析 user_info JSON
   │
   ├─ 填充 Header 结构
   │  ├─ header.UserID = accountInfo.UserID
   │  └─ header.UserEmail = accountInfo.Email
   │
   └─ 存储到 gin.Context
      ├─ c.Set("user_id", accountInfo.UserID)
      ├─ c.Set("user_email", accountInfo.Email)
      ├─ c.Set("user_name", accountInfo.UserName)
      └─ c.Set("account_info", accountInfo)

3. Handler 函数
   └─ 可以从 header 参数中获取用户信息
```

## 💻 使用方法

### 1. 前端发送请求

#### 方式1：使用 Axios 拦截器（推荐）

已在 `frontend/src/utils/request.js` 中实现：

```javascript
request.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    const userEmail = localStorage.getItem('userEmail');
    
    if (token && userEmail) {
      config.headers['Authorization'] = `Bearer ${token}`;
      config.headers['X-User-Email'] = userEmail;
    }
    
    return config;
  }
);
```

#### 方式2：手动设置 Header

```javascript
import axios from 'axios';

axios.post('/api/apps/bms/inventory/create_equip_inv', data, {
  headers: {
    'Authorization': `Bearer ${token}`,
    'X-User-Email': userEmail
  }
});
```

## 🔍 错误处理

### 常见错误及处理

| 错误场景 | 错误码 | 错误信息 | HTTP 状态码 |
|---------|--------|---------|------------|
| 未提供 token | - | "unauthorized: token is required" | 200 (retcode=-1) |
| token 无效或过期 | ErrAuth (-100401) | "unauthorized: token expired or invalid" | 200 (retcode=-100401) |
| Session 不存在 | ErrAuth (-100401) | "unauthorized: token expired or invalid" | 200 (retcode=-100401) |
| 数据库错误 | ErrDB (1) | "database error" | 200 (retcode=1) |

### 前端错误处理

已在 `frontend/src/utils/request.js` 中实现：

```javascript
request.interceptors.response.use(
  (response) => {
    const res = response.data;
    
    // 认证失败，跳转到登录页
    if (res.retcode === -100401 || res.retcode === -100405) {
      message.error('登录已过期，请重新登录');
      localStorage.clear();
      setTimeout(() => {
        window.location.href = '/login';
      }, 1500);
    }
    
    return res;
  }
);
```

## 🔄 Session 管理

### Session 存储结构

**表**: `login_session_tab`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | VARCHAR(64) | 用户ID |
| session_id | VARCHAR(128) | Session ID (格式: `:1:USER_ID:TOKEN前12位`) |
| user_info | TEXT | 用户信息JSON |
| expire_time | BIGINT | 过期时间（Unix时间戳） |
| ctime | BIGINT | 创建时间 |
| mtime | BIGINT | 修改时间 |

### Session 生命周期

```
1. 用户登录
   └─ 生成 32 位 UUID 作为 token
   └─ 创建 session 记录，expire_time = now + 7天
   └─ 返回 token 给前端

2. 前端存储 token
   └─ localStorage.setItem('token', token)

3. 后续请求携带 token
   └─ Authorization: Bearer <token>

4. 后端验证 token
   └─ 查询 login_session_tab
   └─ 检查 expire_time >= now
   └─ 解析 user_info

5. Token 过期
   └─ 返回 ErrAuth 错误
   └─ 前端清除 localStorage
   └─ 跳转到登录页
```

## 📊 性能优化建议

### 1. 添加 Redis 缓存

当前实现每次请求都查询数据库，建议添加 Redis 缓存：

```go
// 伪代码
func (a *AuthService) ValidateToken(ctx context.Context, token string) (*entity.AccountCacheInfo, *bmserror.BMSError) {
    // 1. 先从 Redis 获取
    cacheKey := "session:" + token
    if cached := redis.Get(cacheKey); cached != nil {
        return cached, nil
    }
    
    // 2. Redis 未命中，查询数据库
    accountInfo, err := a.queryFromDB(ctx, token)
    if err != nil {
        return nil, err
    }
    
    // 3. 写入 Redis 缓存，TTL = session 剩余时间
    redis.Set(cacheKey, accountInfo, ttl)
    
    return accountInfo, nil
}
```

### 2. Session 清理任务

定期清理过期的 session 记录：

```sql
-- 删除 7 天前过期的 session
DELETE FROM login_session_tab 
WHERE expire_time < UNIX_TIMESTAMP() - 7*24*3600;
```

## 🎯 安全建议

### 1. Token 安全

- ✅ Token 存储在 localStorage（已实现）
- ⚠️ 建议：使用 HttpOnly Cookie 更安全
- ⚠️ 建议：Token 使用 JWT 格式，包含签名

### 2. HTTPS

- ⚠️ 生产环境必须使用 HTTPS
- ⚠️ 防止 Token 被中间人攻击窃取

### 3. Token 刷新机制

- ⚠️ 建议：实现 refresh token 机制
- ⚠️ Access token 短期有效（1小时）
- ⚠️ Refresh token 长期有效（7天）

### 4. 防止 CSRF

- ✅ 使用 Authorization Header（已实现）
- ✅ 不使用 Cookie 传递 token
