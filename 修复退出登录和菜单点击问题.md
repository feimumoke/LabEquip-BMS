# 修复退出登录和菜单点击问题

## 🐛 问题描述

用户反馈：
1. ❌ 不能退出登录
2. ❌ 菜单点击没反应

---

## 🔍 问题原因

### 问题 1：退出登录失败

**原因**：在 `MainLayout.jsx` 中，Dropdown 菜单项的 `onClick` 事件处理方式不正确。

**错误代码**：
```javascript
const userMenuItems = [
  {
    key: 'logout',
    icon: <LogoutOutlined />,
    label: '退出登录',
    onClick: handleLogout,  // ❌ 在 Ant Design 4.x 中无效
  },
];

<Dropdown menu={{ items: userMenuItems }} placement="bottomRight">
```

**说明**：
- 在 Ant Design 4.x 中，`menu.items` 中的 `onClick` 不会被触发
- 应该在 `menu` 对象上统一定义 `onClick` 处理函数

### 问题 2：菜单点击可能的问题

**可能原因**：
1. 路由配置问题
2. 权限检查问题
3. Navigate 函数未正确执行

## ✅ 修复方案

### 修复 1：退出登录功能

**文件**：`frontend/src/components/Layout/MainLayout.jsx`

#### 修改前：
```javascript
const userMenuItems = [
  {
    key: 'logout',
    icon: <LogoutOutlined />,
    label: '退出登录',
    onClick: handleLogout,  // ❌ 无效
  },
];

<Dropdown menu={{ items: userMenuItems }} placement="bottomRight">
```

#### 修改后：
```javascript
const userMenuItems = [
  {
    key: 'logout',
    icon: <LogoutOutlined />,
    label: '退出登录',
  },
];

// ✅ 添加统一的菜单点击处理函数
const handleUserMenuClick = ({ key }) => {
  if (key === 'logout') {
    handleLogout();
  } else if (key === 'profile') {
    message.info('个人信息功能待开发');
  }
};

// ✅ 在 menu 对象上添加 onClick
<Dropdown menu={{ items: userMenuItems, onClick: handleUserMenuClick }} placement="bottomRight">
```

### 问题 B：菜单点击没反应

**调试步骤**：

1. 打开浏览器控制台
2. 运行以下代码：
```javascript
// 检查 navigate 函数
import { useNavigate } from 'react-router-dom';
const navigate = useNavigate();
console.log('navigate:', navigate);

// 手动测试导航
navigate('/equipment');
```

3. 如果手动导航工作，说明问题在 Menu 组件

**可能的问题**：

#### 问题 B1：Menu onClick 未触发

**检查**：在 `MainLayout.jsx` 中添加 console.log：

```javascript
<Menu
  theme="dark"
  selectedKeys={[location.pathname]}
  mode="inline"
  items={getMenuItems()}
  onClick={({ key }) => {
    console.log('Menu clicked, key:', key);  // 添加这行
    navigate(key);
  }}
/>
```

如果点击菜单没有输出日志，说明：
- Menu 组件被其他元素覆盖
- 事件被拦截
- CSS 样式问题（如 `pointer-events: none`）

**解决方案**：
```javascript
// 检查 Menu 的样式
<Menu
  style={{ pointerEvents: 'auto' }}  // 确保可以点击
  theme="dark"
  selectedKeys={[location.pathname]}
  mode="inline"
  items={getMenuItems()}
  onClick={({ key }) => navigate(key)}
/>
```

#### 问题 B2：路由未正确匹配

**检查**：访问 `http://localhost:3000/equipment`，查看是否显示正确的页面。

如果直接访问 URL 能工作，但点击菜单不工作，说明：
- Menu 的 key 与路由路径不匹配
- onClick 中的 navigate 逻辑有问题

**解决方案**：确保 Menu items 的 key 与路由路径完全一致：

```javascript
const getMenuItems = () => {
  return [
    {
      key: '/equipment',  // ✅ 必须以 / 开头，与路由路径一致
      icon: <ToolOutlined />,
      label: '设备管理',
    },
    // ...
  ];
};
```

#### 问题 B3：权限检查阻止导航

**检查**：查看控制台是否有权限相关的错误。

**解决方案**：暂时注释掉权限检查：

```javascript
// 临时测试
<Route path="inventory/manage" element={<InventoryManage />} />
// 而不是
<Route path="inventory/manage" element={
  <PermissionCheck requireAdmin>
    <InventoryManage />
  </PermissionCheck>
} />
```
