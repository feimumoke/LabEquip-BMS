# 日志系统快速参考

## 📝 配置（server/_config/conf.yaml）

```yaml
log:
  dir: "./logs"           # 日志目录
  level: "INFO"          # DEBUG|INFO|WARN|ERROR|FATAL
  enableConsole: true    # 是否输出到控制台
  gormLogLevel: "INFO"   # SILENT|ERROR|WARN|INFO (SQL日志)
```

---

## 💻 使用方法

```go
import "github.com/feimumoke/labequipbms/framework/log"

// 基本使用（不带 RequestID）
log.Infof("Server started on port %d\n", 8080)
log.Debugf("Debug info: %v\n", data)
log.Warnf("Warning: %s\n", msg)
log.Errorf("Error: %v\n", err)
log.Fatalf("Fatal: %v\n", err)  // 会退出程序

// 带上下文（推荐，自动包含 RequestID）
log.CtxInfof(ctx, "User %s logged in\n", username)
log.CtxDebugf(ctx, "Debug: %v\n", data)
log.CtxWarnf(ctx, "Warning: %s\n", msg)
log.CtxErrorf(ctx, "Failed: %v\n", err)
```

## 📂 日志文件

**位置**：`logs/bms-YYYY-MM-DD.log`

**示例**：
```
logs/
├── bms-2026-01-15.log  # 昨天
├── bms-2026-01-16.log  # 今天
└── bms-2026-01-17.log  # 明天
```

**特性**：
- ✅ 自动按天切割
- ✅ 自动清理 7 天前的日志
- ✅ 同时输出到控制台（可配置）

## 🔍 查看日志

```bash
# 实时查看今天的日志
tail -f logs/bms-$(date +%Y-%m-%d).log

# 查看最近 100 行
tail -n 100 logs/bms-$(date +%Y-%m-%d).log

# 查看错误日志
grep "\[ERROR\]" logs/bms-$(date +%Y-%m-%d).log

# 查看警告和错误
grep -E "\[WARN\]|\[ERROR\]" logs/bms-$(date +%Y-%m-%d).log

# 查看 SQL 日志
grep "\[GORM\]" logs/bms-$(date +%Y-%m-%d).log

# 查看慢查询
grep "SLOW SQL" logs/bms-$(date +%Y-%m-%d).log

# 根据 RequestID 追踪请求链路
grep "123e4567-e89b-12d3-a456-426614174000" logs/bms-$(date +%Y-%m-%d).log

# 搜索关键词
grep "database" logs/bms-$(date +%Y-%m-%d).log
```

## ⚙️ 不同环境配置

### 开发环境
```yaml
log:
  dir: "./logs"
  level: "DEBUG"          # 显示所有日志
  enableConsole: true     # 输出到控制台
  gormLogLevel: "INFO"    # 显示所有 SQL
```

### 生产环境
```yaml
log:
  dir: "/var/log/bms"     # 系统日志目录
  level: "INFO"           # 只记录重要信息
  enableConsole: false    # 不输出到控制台
  gormLogLevel: "ERROR"   # 只记录 SQL 错误
```

## 🔧 问题排查

### 日志文件未创建
```bash
# 检查目录
ls -la logs/

# 检查权限
ls -ld logs/

# 检查配置
cat server/_config/conf.yaml | grep -A 3 "log:"
```

### 日志未输出到控制台
```yaml
# 确保配置为 true
log:
  enableConsole: true
```

### 日志文件过大
```bash
# 查看文件大小
du -h logs/*.log

# 手动清理旧日志
find logs/ -name "bms-*.log" -mtime +7 -delete
```

## 📊 日志级别

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| DEBUG | 调试信息 | `log.Debugf("Params: %+v\n", params)` |
| INFO | 重要操作 | `log.Infof("User login: %s\n", user)` |
| WARN | 警告信息 | `log.Warnf("Cache miss: %s\n", key)` |
| ERROR | 错误信息 | `log.Errorf("DB error: %v\n", err)` |
| FATAL | 致命错误 | `log.Fatalf("Cannot start: %v\n", err)` |

## ✅ 最佳实践

### ✅ 好的日志
```go
// 推荐：使用 CtxXXXf 自动包含 RequestID
log.CtxInfof(ctx, "User login success, user_id=%s, ip=%s\n", userID, ip)
log.CtxErrorf(ctx, "Query failed, table=%s, error=%v\n", table, err)

// 一般：不需要追踪链路的系统日志
log.Infof("Server started on port %d\n", port)
```

### ❌ 不好的日志
```go
log.Infof("success\n")               // 信息不足
log.Errorf("error: %v\n", err)       // 缺少上下文
log.Infof("password: %s\n", pwd)     // 敏感信息
log.Infof(ctx, "login\n")            // 应该用 CtxInfof
```

## 🆔 TraceID 追踪

每个 HTTP 请求自动生成唯一的 TraceID，用于追踪请求链路。

### 日志格式
```
[INFO] [TraceID: 123e4567-e89b-12d3-a456-426614174000] User login successful
[GORM] [TraceID: 123e4567-e89b-12d3-a456-426614174000] SELECT * FROM user_tab
```

### 使用方式

#### HTTP 请求（自动）
```go
// 在 Handler 中获取 context
func UserHandler(c *gin.Context) {
    ctx := c.Request.Context()
    
    // 使用 CtxXXXf 自动包含 TraceID
    log.CtxInfof(ctx, "Processing request\n")
    
    // 传递给 Service 层
    result, err := userService.Process(ctx, data)
}
```

#### 定时任务（手动）
```go
func CronTask() {
    ctx := context.Background()
    traceID := log.GetOrNewTraceID(ctx)
    ctx = context.WithValue(ctx, "trace_id", traceID)
    
    log.CtxInfof(ctx, "Cron task started\n")
}
```

### 核心函数
```go
// 获取或创建 TraceID（推荐）
traceID := log.GetOrNewTraceID(ctx)
traceID := web.GetOrNewTraceID(ctx)

// 仅获取 TraceID
traceID := web.GetTraceID(ctx)
traceID := web.GetTraceIDFromGin(c)
```

### 追踪请求链路
```bash
# 根据 TraceID 查找所有相关日志
grep "123e4567-e89b-12d3-a456-426614174000" logs/bms-$(date +%Y-%m-%d).log

# 实时监控特定请求
tail -f logs/bms-$(date +%Y-%m-%d).log | grep "123e4567-e89b-12d3-a456-426614174000"

# 查看所有带 TraceID 的日志
grep "\[TraceID:" logs/bms-$(date +%Y-%m-%d).log
```
