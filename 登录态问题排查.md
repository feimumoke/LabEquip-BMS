# 登录态校验问题排查和解决

## ❌ 问题描述

前端收到错误：
```json
{"retcode":-1,"message":"unauthorized: token is required","data":null}
```

## 🔍 问题原因

### 原因 1：路径匹配问题

- **前端配置**: baseURL = `/api`，所以实际请求是 `/api/apps/common/enums`
- **后端白名单**: 检查路径是否以 `/apps/common/enums` 开头
- **结果**: 不匹配！`/api/apps/common/enums` 不是以 `/apps/common/enums` 开头

### 原因 2：应用启动时加载枚举

前端在 `App.js` 的 `useEffect` 中尝试加载枚举，但这时用户可能还未登录。

## ✅ 解决方案

### 1. 修改后端白名单匹配逻辑

**文件**: `framework/web/auth_service.go`

**修改前**:
```go
func IsWhiteListPath(path string) bool {
    for _, whitePath := range whiteListPaths {
        if strings.HasPrefix(path, whitePath) {
            return true
        }
    }
    return false
}
```

**修改后**:
```go
func IsWhiteListPath(path string) bool {
    for _, whitePath := range whiteListPaths {
        // 使用 Contains 而不是 HasPrefix
        // 因为路径可能包含前缀如 /api
        if strings.Contains(path, whitePath) {
            return true
        }
    }
    return false
}
```

### 2. 移除应用启动时的枚举加载

**文件**: `frontend/src/App.js`

**修改前**:
```javascript
useEffect(() => {
  if (isAuthenticated()) {
    enumStore.loadEnums();
  }
}, []);
```

**修改后**:
```javascript
// 注释掉应用启动时的枚举加载
// 枚举只在登录成功后加载
```

### 3. 优化前端请求拦截器

**文件**: `frontend/src/utils/request.js`

**添加前端白名单检查**:
```javascript
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  const userEmail = localStorage.getItem('userEmail');
  
  // 白名单接口：不需要 token
  const whiteListUrls = [
    '/apps/basic/user/user_login',
    '/apps/basic/user/create_user',
    '/apps/common/enums',
  ];
  
  const isWhiteList = whiteListUrls.some(url => config.url.includes(url));
  
  // 只有非白名单接口且有 token 时才添加认证头
  if (!isWhiteList && token && userEmail) {
    config.headers['Authorization'] = `Bearer ${token}`;
    config.headers['X-User-Email'] = userEmail;
  }
  
  return config;
});
```

## 🎯 最佳实践

### 枚举加载时机

**推荐做法**：只在登录成功后加载枚举

```javascript
// Login/index.jsx
const onLoginFinish = async (values) => {
  const res = await login({...});
  
  if (res && res.retcode === 0) {
    // ... 保存用户信息
    
    // 登录成功后立即加载枚举
    enumStore.loadEnums();
    
    navigate('/inventory/list');
  }
};
```

**不推荐**：在应用启动时加载
- ❌ 用户可能未登录
- ❌ Token 可能已过期
- ❌ 会导致不必要的错误请求

## 📝 常见问题

### Q1: 为什么改用 Contains 而不是 HasPrefix？

**A**: 因为实际请求路径可能包含前缀。

例如：
- Nginx 配置了 `/api` 前缀
- 实际路径: `/api/apps/common/enums`
- 白名单路径: `/apps/common/enums`
- `HasPrefix("/api/apps/common/enums", "/apps/common/enums")` = false ❌
- `Contains("/api/apps/common/enums", "/apps/common/enums")` = true ✅

### Q2: 前后端都需要白名单吗？

**A**: 是的，各有作用：

- **前端白名单**: 避免发送不必要的 token
- **后端白名单**: 安全验证，防止未授权访问

### Q3: 如何添加新的白名单路径？

**A**: 需要同时修改前后端：

1. **后端**: `framework/web/auth_service.go`
   ```go
   var whiteListPaths = []string{
       "/your/new/path",
   }
   ```

2. **前端**: `frontend/src/utils/request.js`
   ```javascript
   const whiteListUrls = [
       '/your/new/path',
   ];
   ```

### Q4: 枚举加载失败怎么办？

**A**: 使用降级处理：

```javascript
// enumStore.js
async loadEnums() {
  try {
    const res = await getEnums();
    if (res && res.retcode === 0 && res.data) {
      this.enums = res.data;
      this.isLoaded = true;
    }
  } catch (error) {
    console.error('Failed to load enums:', error);
    // 使用默认枚举或空对象
    this.enums = {};
  }
}
```
