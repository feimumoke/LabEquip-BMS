# 登录态保持修复说明

## 📋 问题描述

原来的登录接口 `user_login` 只返回了 `username`，没有返回 **session token** 和**用户详细信息**，导致前端无法保持登录态。

## ✅ 修复内容

### 1. **修改 Proto 定义**

#### 文件: `api_idl/apps/basic/user_login.proto` 和 `user_login.tpl.proto`

添加了 `UserInfo` 消息和扩展 `UserLoginResponse`：

```protobuf
message UserInfo {
  required string user_id = 1;
  required string name = 2;
  required string email = 3;
  required int64 role = 4;
  optional string phone = 5;
  optional string avatar = 6;
}

message UserLoginResponse{
  required int64 login_status = 1;
  optional string user_name = 2;
  optional string token = 3;           // 新增：session token
  optional UserInfo user_info = 4;     // 新增：用户详细信息
}
```

### 2. **更新 Protobuf 生成代码**

#### 文件: `api_idl/apps/basic/user_login.pb.go`

手动添加了：
- `UserInfo` 结构体及其 Getter 方法
- `UserLoginResponse` 的 `Token` 和 `UserInfo` 字段
- 相应的 `GetToken()` 和 `GetUserInfo()` 方法

### 3. **修改后端 Handler**

#### 文件: `apps/basic/view/user_handler.go`

修改 `UserLoginHandler` 返回完整的用户信息和 session token：

```go
loginStatus := int64(1)
userName := user.Name

// 构造用户信息
userInfo := &pbbasic.UserInfo{
    UserId: &user.UserID,
    Name:   &user.Name,
    Email:  &user.Email,
    Role:   (*int64)(&user.Role),
    Phone:  &user.Phone,
    Avatar: &user.Avatar,

// 返回 session token
token := sessionID

return &pbbasic.UserLoginResponse{
    LoginStatus: &loginStatus,
    UserName:    &userName,
    Token:       &token,
    UserInfo:    userInfo,
}, nil
```

### 4. **修改前端 API 调用**

### 5. **前端登录态处理**

#### 文件: `frontend/src/pages/Login/index.jsx`

前端已正确处理返回的数据：

```javascript
const res = await login({
  email: values.email,
  password: values.password,
});

if (res && res.retcode === 0) {
  const data = res.data || res;
  const token = data.token;        // 获取 session token
  const userInfo = data.user_info; // 获取用户信息
  
  saveAuthInfo(token, values.email, userInfo); // 保存到 localStorage
  authStore.setUser(userInfo);                 // 保存到 MobX store
  message.success('登录成功');
  navigate('/');
}
```

## 🔐 登录态机制

### 后端

1. **Session 创建**: 用户登录成功后，后端创建 `LoginSession` 记录，包含：
   - `SessionID`: 32位唯一标识符
   - `UserInfo`: JSON格式的用户信息
   - `ExpireTime`: 7天后过期

2. **Session 返回**: 将 `SessionID` 作为 `token` 返回给前端

3. **Session 验证**: 前端后续请求携带 token，后端通过 `LoginStatusInterceptor` 验证session有效性

### 前端

1. **Token 保存**: 将 token 保存到 `localStorage.token`
2. **用户信息保存**: 将 user_info 保存到 `localStorage.userInfo`
3. **邮箱保存**: 将 userEmail 保存到 `localStorage.userEmail`

4. **请求携带**: 
   - 请求头 `Authorization: Bearer {token}`
   - 请求头 `X-User-Email: {userEmail}`

5. **权限检查**: 
   - `isAuthenticated()`: 检查是否登录
   - `isTeacher()`: 检查是否是教师
   - `isAdmin()`: 检查是否是管理员

## 🔧 故障排查

### 问题1: 登录后立即退出

**原因**: Token 未保存或保存失败

**解决**: 
1. 检查浏览器控制台是否有错误
2. 检查 LocalStorage 是否有 token
3. 检查后端是否正确返回 token

### 问题2: 刷新页面后退出登录

**原因**: Token 验证失败

**解决**:
1. 检查后端 `LoginStatusInterceptor` 是否正常工作
2. 检查 session 是否过期
3. 检查请求头是否正确携带 token

### 问题3: 权限不足

**原因**: 用户角色不正确

**解决**:
1. 检查 `user_info.role` 值是否正确
2. 检查权限判断逻辑是否正确
3. 检查后端权限拦截器配置
