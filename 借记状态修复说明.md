# 借记状态修复说明

## 🐛 问题描述

1. **审批管理页面**：查询的状态是 2（已审批），但应该查询状态 1（待审批）
2. **所有借记页面**：使用了错误的状态映射，与实际的 `BorrowTaskStatus` 枚举不一致

---

## ✅ 修复内容

### 1. 审批管理页面 (`frontend/src/pages/Borrow/Approve.jsx`)

#### 修复前
```javascript
const res = await searchBorrowTask({
  task_status: 2, // ❌ 错误：2 是"已审批"状态
});
```

#### 修复后
```javascript
const res = await searchBorrowTask({
  task_status: 1, // ✅ 正确：1 是"待审批"状态
});
```

#### 其他改进
- ✅ 引入 `enumStore` 使用枚举显示状态
- ✅ 使用 `observer` 包装组件实现响应式
- ✅ 状态标签使用枚举文本而不是硬编码

---

### 2. 所有借记页面 (`frontend/src/pages/Borrow/AllTasks.jsx`)

#### 修复前（错误的状态映射）
```javascript
const taskStatusMap = {
  1: { text: '待分配', color: 'default' },      // ❌ 错误
  2: { text: '已分配', color: 'processing' },   // ❌ 错误
  3: { text: '已审批', color: 'success' },      // ❌ 错误
  4: { text: '已完成', color: 'success' },      // ❌ 错误
  5: { text: '已取消', color: 'error' },        // ❌ 错误
  6: { text: '已拒绝', color: 'error' },        // ❌ 错误
};
```

#### 修复后（使用正确的枚举）
```javascript
// 使用 enumStore 获取状态文本
const label = enumStore.getEnumLabel('BorrowTaskStatus', status);

// 正确的颜色映射
const getStatusColor = (status) => {
  const colorMap = {
    1: 'orange',    // 待审批
    2: 'blue',      // 已审批
    3: 'cyan',      // 已分配
    4: 'green',     // 进行中
    7: 'red',       // 已拒绝
    8: 'default',   // 已归还
    9: 'default',   // 已取消
  };
  return colorMap[status] || 'default';
};
```

#### 其他改进
- ✅ 使用 `EnumSelect` 组件替代手动编写的 Select
- ✅ 状态筛选器自动从枚举获取选项
- ✅ 添加页面标题和描述
- ✅ 改进 UI 布局，使用 Card 组件

## 📊 正确的状态定义

### BorrowTaskStatus 枚举值

| 值 | 常量名 | 中文名称 | 说明 | 颜色 |
|----|--------|---------|------|------|
| 1 | BorrowTaskStatusPending | 待审批 | 创建了，没分配库存 | 🟠 橙色 |

| 3 | BorrowTaskStatusAllocate | 已分配 | 已分配库存 | 🔷 青色 |
| 4 | BorrowTaskStatusOngoing | 进行中 | 已拿走 | 🟢 绿色 |
| 7 | BorrowTaskStatusReject | 已拒绝 | 审批拒绝 | 🔴 红色 |
| 8 | BorrowTaskStatusDone | 已归还 | 归还 | ⚪ 灰色 |
| 9 | BorrowTaskStatusCancel | 已取消 | 取消 | ⚪ 灰色 |

## 🔄 状态流转逻辑

### 正常流程

```
1. 待审批 (Pending, 1) ← 【审批管理页面应该查询这个状态】
   ├─ 学生创建借记任务
   └─ 等待教师审批
   
   ↓ (教师审批通过)
   
2. 已审批 (Approval, 2)
   ├─ 审批通过
   └─ 系统分配库存
   
   ↓ (库存分配成功)
   
3. 已分配 (Allocate, 3)
   ├─ 库存已分配
   └─ 学生可以拿走设备
   
   ↓ (学生拿走设备)
   
4. 进行中 (Ongoing, 4)
   ├─ 学生已拿走设备
   └─ 等待归还
   
   ↓ (学生归还设备)
   
8. 已归还 (Done, 8)
   └─ 任务完成
```

### 异常流程

```
待审批 (1) → 已取消 (9)
   └─ 学生主动取消

已审批 (2) → 已拒绝 (7)
   └─ 教师拒绝审批
```

## 📝 注意事项

### 1. 状态值不连续

状态值为 1, 2, 3, 4, 7, 8, 9，**不是连续的**！

- ❌ 不要假设状态值是 1-6 或 1-9 连续的
- ✅ 使用枚举定义的实际值

### 2. 前后端保持一致

- ✅ 后端定义：`defines/constant/borrow_enum.go`
- ✅ 前端获取：通过 `/api/apps/common/enums` 接口
- ✅ 前端使用：`enumStore.getEnumLabel('BorrowTaskStatus', value)`

### 3. 审批流程

```
待审批 (1) → 教师审批 → 已审批 (2) 或 已拒绝 (7)
```

**重要**: 审批管理页面应该查询状态 1（待审批），而不是状态 2（已审批）！
