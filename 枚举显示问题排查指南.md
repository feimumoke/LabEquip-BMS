## 🔧 可能的问题和解决方案

### 问题 1：枚举未加载

**症状**：
- `enumStore.isLoaded` 为 `false`
- `enumStore.enums` 为空对象 `{}`
- 所有状态都显示数字

**原因**：
- 登录后没有调用 `enumStore.loadEnums()`
- `/api/apps/common/enums` 接口请求失败

**解决方案**：

#### 步骤 1：确认登录后加载枚举

检查 `frontend/src/pages/Login/index.jsx`：

```javascript
const handleLogin = async (values) => {
  try {
    const res = await login({ ... });

    if (res.retcode === 0) {
      authStore.setUser(res.data.user_info);
      setToken(res.data.token);
      setUserEmail(res.data.user_info.email);
      setUserInfo(JSON.stringify(res.data.user_info));

      // ✅ 确保这行代码存在
      await enumStore.loadEnums();

      message.success('登录成功');
      navigate('/inventory/list');
    }
  } catch (error) {
    message.error('登录失败');
  }
};
```

#### 步骤 2：检查网络请求

在浏览器的 Network 标签中，找到 `/api/apps/common/enums` 请求：

1. **如果请求失败（401/403）**：
   - 检查 token 是否正确保存
   - 检查白名单配置（该接口应该不需要 token）

2. **如果请求成功但返回空数据**：
   - 检查后端是否正确注册了枚举
   - 重启后端服务

### 问题 2：BorrowTaskStatus 枚举未注册

**症状**：
- 其他枚举（如 EquipCategory）显示正常
- 只有 BorrowTaskStatus 显示数字
- 控制台警告：`⚠️ Enum 'BorrowTaskStatus' not found`

**原因**：
- 后端没有注册 `BorrowTaskStatus` 枚举
- `init()` 函数没有被调用

**解决方案**：

#### 步骤 1：检查后端枚举定义

文件：`defines/constant/borrow_enum.go`

```go
// 导出给前端的借记任务状态枚举
var ExportBorrowTaskStatusNameToValue = map[string]interface{}{
	"待审批": BorrowTaskStatusPending,  // 1
	"已审批": BorrowTaskStatusApproval, // 2
	"已分配": BorrowTaskStatusAllocate, // 3
	"进行中": BorrowTaskStatusOngoing,  // 4
	"已拒绝": BorrowTaskStatusReject,   // 7
	"已归还": BorrowTaskStatusDone,     // 8
	"已取消": BorrowTaskStatusCancel,   // 9
}

// ✅ 确保 init() 函数存在
func init() {
	RegisterEnumValues("BorrowTaskStatus", ExportBorrowTaskStatusNameToValue)
}
```

#### 步骤 2：确认包被导入

检查 `main.go` 或其他初始化文件，确保 `borrow_enum.go` 所在的包被导入：

```go
import (
	_ "github.com/feimumoke/labequipbms/defines/constant" // 确保 init() 被执行
)
```

#### 步骤 3：重启后端服务

```bash
# 停止后端
# 重新编译并启动
go run main.go
```

### 问题 3：枚举值类型不匹配

**症状**：
- 枚举已加载
- `BorrowTaskStatus` 枚举存在
- 但状态仍显示数字
- 控制台警告：`⚠️ Value 1 not found in enum 'BorrowTaskStatus'`

**原因**：
- 后端返回的值类型与前端比较的类型不一致
- 例如：后端返回字符串 `"1"`，前端比较的是数字 `1`

**解决方案**：

#### 步骤 1：检查后端返回的数据类型

在 Network 标签中查看 `/api/apps/common/enums` 的响应：

```json
{
  "retcode": 0,
  "data": {
    "BorrowTaskStatus": {
      "待审批": 1,      // ✅ 正确：数字类型
      "已审批": 2,
      ...
    }
  }
}
```

**错误示例**：
```json
{
  "BorrowTaskStatus": {
    "待审批": "1",     // ❌ 错误：字符串类型
    "已审批": "2",
    ...
  }
}
```

#### 步骤 2：检查后端数据库返回的类型

查看 `apps/bms/view/borrowing_handler.go` 中 `SearchBorrowTaskHandler`：

```go
// 确保 task_status 是 int64 类型
task.TaskStatus = borrowTask.TaskStatus // int64
```

#### 步骤 3：增强前端类型容错

如果后端确实返回字符串，可以修改 `enumStore.getEnumLabel()`：

```javascript
getEnumLabel(enumKey, value) {
  const enumData = this.getEnum(enumKey);
  
  // 如果是对象格式，查找对应的key
  for (const [label, val] of Object.entries(enumData)) {
    // ✅ 增加类型转换，支持字符串和数字的比较
    if (val == value || String(val) === String(value)) {
      return label;
    }
  }
  
  return String(value);
}
```

### 问题 4：前端缓存问题

**症状**：
- 后端数据正确
- 控制台显示枚举已加载
- 但页面上仍显示数字
- 刷新页面后正常

**原因**：
- React 组件没有响应 enumStore 的变化
- 组件没有用 `observer` 包装

**解决方案**：

#### 步骤 1：确保组件用 observer 包装

检查所有使用 `enumStore` 的组件：

```javascript
import { observer } from 'mobx-react-lite';
import enumStore from '../../store/enumStore';

// ✅ 正确：用 observer 包装
const MyTasks = observer(() => {
  // ...
  const label = enumStore.getEnumLabel('BorrowTaskStatus', status);
  // ...
});

export default MyTasks;
```

**错误示例**：
```javascript
// ❌ 错误：没有用 observer 包装
const MyTasks = () => {
  const label = enumStore.getEnumLabel('BorrowTaskStatus', status);
  // 组件不会响应 enumStore 的变化
};
```

#### 步骤 2：清除浏览器缓存

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

或者：

1. 打开浏览器设置
2. 清除浏览器数据
3. 选择"缓存的图片和文件"
4. 清除 LocalStorage：
   ```javascript
   localStorage.clear();
   ```
