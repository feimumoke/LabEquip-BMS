# 前端启动步骤（已降级版本）

## 版本说明

已将依赖降级为兼容 Node.js 12.x/14.x 的版本：

- React: 17.0.2（降级）
- Ant Design: 4.24.0（降级）
- react-scripts: 4.0.3（降级）
- Node.js: 建议 14.21.3

## 快速启动

### 1. 进入前端目录
```bash
cd frontend
```

### 2. 删除旧依赖（重要！）
```bash
rm -rf node_modules package-lock.json
```

### 3. 安装依赖
```bash
npm install
```

如果遇到依赖冲突，使用强制安装：
```bash
npm install --legacy-peer-deps
```

### 4. 启动开发服务器
```bash
npm start
```

项目将在 http://localhost:3000 启动

## 常见问题解决

### 问题 1: 依赖安装失败

**解决方案：**
```bash
# 清除缓存
npm cache clean --force

# 使用强制安装
npm install --legacy-peer-deps
```

### 问题 2: 端口被占用

**解决方案：**
```bash
# 使用其他端口
PORT=3001 npm start
```

### 问题 3: 编译错误

**解决方案：**
```bash
# 删除 build 缓存
rm -rf build

# 重新启动
npm start
```

### 问题 4: ESLint 错误

如果还有 ESLint 相关错误，可以临时禁用：

创建 `.env` 文件：
```bash
echo "ESLINT_NO_DEV_ERRORS=true" > .env
echo "DISABLE_ESLINT_PLUGIN=true" >> .env
```

## 完整的清理和重装流程

如果遇到任何问题，执行完整清理：

```bash
cd frontend

# 1. 删除所有依赖和缓存
rm -rf node_modules package-lock.json build

# 2. 清除 npm 缓存
npm cache clean --force

# 3. 重新安装（使用强制安装避免依赖冲突）
npm install --legacy-peer-deps

# 4. 启动
npm start
```

## 环境要求

- Node.js: 12.x 或 14.x（推荐 14.21.3）
- npm: 6.x 或 8.x
- 操作系统: macOS/Linux/Windows

## 检查环境

```bash
node -v   # 应该显示 v12.x 或 v14.x
npm -v    # 应该显示 6.x 或 8.x
```

## 如果需要切换 Node.js 版本

使用 nvm 切换：

```bash
# 安装 nvm（如果还没安装）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 安装 Node.js 14
nvm install 14

# 使用 Node.js 14
nvm use 14

# 验证版本
node -v
```

## 版本降级说明

### 主要变化：

1. **React 18 → 17**
   - 移除了自动批处理
   - `createRoot` 改为 `ReactDOM.render`
   
2. **Ant Design 5 → 4**
   - Modal 的 `open` 改为 `visible`
   - Tabs 的 `TabPane` 改为 `Tabs.TabPane`
   - 部分 API 有变化

3. **react-scripts 5 → 4**
   - 兼容较低版本的 Node.js
   - 移除了 `node:` 协议依赖

### API 差异已修复：

✅ Modal 组件：`open` → `visible`
✅ Tabs 组件：`TabPane` → `Tabs.TabPane`
✅ 所有其他组件已适配

## 构建生产版本

```bash
npm run build
```

生产文件将生成在 `build` 目录。

## 技术支持

如果仍然遇到问题：

1. 确认 Node.js 版本在 12.x-14.x 范围
2. 删除所有依赖重新安装
3. 使用 `--legacy-peer-deps` 参数
4. 检查是否有防火墙或代理问题
