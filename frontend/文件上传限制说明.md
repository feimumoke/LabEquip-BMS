修改了文件上传组件，现在只允许上传图片和视频文件，其他文件类型在文件选择对话框中会被置灰（不可选）。

### 2. **上传前校验增强**

#### 修改前 ❌

```javascript
const beforeUpload = (file) => {
  const isImage = file.type.startsWith('image/');
  if (!isImage) {
    message.error('只能上传图片文件！');
    return false;
  }

  const isLt10M = file.size / 1024 / 1024 < 10;
  if (!isLt10M) {
    message.error('图片大小不能超过 10MB！');
    return false;
  }

  return true;
};
```

#### 修改后 ✅

```javascript
const beforeUpload = (file) => {
  const isImage = file.type.startsWith('image/');
  const isVideo = file.type.startsWith('video/');
  
  // 检查文件类型
  if (!isImage && !isVideo) {
    message.error('只能上传图片或视频文件！');
    return false;
  }

  // 根据文件类型设置不同的大小限制
  const maxSize = isVideo ? 100 : 10; // 视频100MB，图片10MB
  const isValidSize = file.size / 1024 / 1024 < maxSize;
  
  if (!isValidSize) {
    message.error(`${isVideo ? '视频' : '图片'}大小不能超过 ${maxSize}MB！`);
    return false;
  }

  return true;
};
```

**改进点**：
- ✅ 支持视频文件上传
- ✅ 图片和视频有不同的大小限制
- ✅ 错误提示更友好，明确指出是图片还是视频

## 🎯 工作原理

### 1. **浏览器原生限制（accept 属性）**

```jsx
accept="image/*,video/*"
```

**作用**：
- 在文件选择对话框中，只显示符合条件的文件
- 其他文件类型会被置灰（不可选）
- 这是浏览器原生行为，无需额外代码

**支持的语法**：
- `image/*` - 所有图片类型
- `video/*` - 所有视频类型
- `image/png,image/jpeg` - 指定具体格式
- `.jpg,.png,.mp4` - 按文件扩展名

### 2. **JavaScript 二次校验（beforeUpload）**

虽然 `accept` 属性可以限制文件选择，但用户仍可能通过其他方式（如拖拽）上传不符合要求的文件，因此需要在 JavaScript 中进行二次校验。

```javascript
const beforeUpload = (file) => {
  // 检查 MIME 类型
  const isImage = file.type.startsWith('image/');
  const isVideo = file.type.startsWith('video/');
  
  if (!isImage && !isVideo) {
    message.error('只能上传图片或视频文件！');
    return false;  // 阻止上传
  }
  
  // 检查文件大小
  const maxSize = isVideo ? 100 : 10;
  const isValidSize = file.size / 1024 / 1024 < maxSize;
  
  if (!isValidSize) {
    message.error(`文件大小不能超过 ${maxSize}MB！`);
    return false;  // 阻止上传
  }
  
  return true;  // 允许上传
};
```

## ⚠️ 注意事项

### 1. **浏览器兼容性**

`accept` 属性在所有现代浏览器中都支持，但具体行为可能略有差异：

| 浏览器 | 支持情况 | 说明 |
|--------|---------|------|
| Chrome | ✅ 完全支持 | 文件对话框中只显示匹配的文件 |
| Firefox | ✅ 完全支持 | 文件对话框中只显示匹配的文件 |
| Safari | ✅ 完全支持 | 文件对话框中只显示匹配的文件 |
| Edge | ✅ 完全支持 | 文件对话框中只显示匹配的文件 |
| IE11 | ⚠️ 部分支持 | 支持 accept，但过滤不够严格 |

### 2. **安全性**

- ⚠️ `accept` 属性只是前端限制，可以被绕过
- ✅ 必须在后端也进行文件类型和大小校验
- ✅ 建议后端检查文件的魔数（Magic Number）而不仅仅是扩展名

### 3. **用户体验**

- ✅ 提供清晰的错误提示
- ✅ 在界面上明确说明支持的文件类型和大小限制
- ✅ 考虑添加文件格式图标，让用户更直观
