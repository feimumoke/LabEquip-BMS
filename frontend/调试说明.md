# 前端调试说明

## 登录问题排查步骤

### 1. 打开浏览器开发者工具

按 `F12` 或右键 → 检查，打开开发者工具

### 2. 查看控制台（Console）

点击登录按钮后，查看控制台输出：

**正常情况应该看到：**
```
登录表单提交: {email: "xxx", password: "xxx"}
开始调用登录 API...
发送请求: POST /api/apps/basic/user/user_login {...}
收到响应: /api/apps/basic/user/user_login 200 {...}
登录 API 响应: {...}
保存用户信息: {...}
```

**异常情况：**
- 如果没有任何输出 → 表单没有提交，检查按钮是否绑定正确
- 如果有 "发送请求" 但没有 "收到响应" → 后端没有响应
- 如果有错误信息 → 根据错误内容排查

### 3. 查看网络（Network）标签

1. 切换到 Network 标签
2. 点击登录按钮
3. 查找 `user_login` 请求

**检查项：**
- **Status**: 应该是 200
- **Request URL**: 应该是 `http://localhost:3000/api/apps/basic/user/user_login`
- **Request Method**: POST
- **Request Payload**: 包含 email 和 password
- **Response**: 查看响应内容

### 4. 常见问题及解决方案

#### 问题 1: 404 Not Found

**原因**: 后端服务未启动或 API 路径不对

**解决方案**:
```bash
# 1. 检查后端是否运行
# 2. 检查后端监听端口是否是 8080
# 3. 检查 API 路径是否正确
```

#### 问题 2: CORS 错误

**错误信息**:
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解决方案**: 后端需要配置 CORS，允许前端访问

#### 问题 3: 网络错误 (ERR_CONNECTION_REFUSED)

**原因**: 无法连接到后端服务器

**解决方案**:
```bash
# 1. 确认后端服务已启动
# 2. 确认后端监听在 localhost:8080
# 3. 检查防火墙设置
```

#### 问题 4: 500 服务器错误

**原因**: 后端处理请求时出错

**解决方案**: 查看后端日志，检查数据库连接、请求参数等

#### 问题 5: 响应格式不对

**错误信息**: "登录响应格式错误"

**原因**: 后端返回的数据格式与前端期望不一致

**期望的响应格式**:
```json
{
  "code": 200,
  "data": {
    "token": "xxx",
    "user_info": {
      "email": "xxx",
      "name": "xxx",
      "role": 1
    }
  }
}
```

### 5. 手动测试后端 API

使用 curl 命令测试：

```bash
# 测试登录接口
curl -X POST http://localhost:8080/apps/basic/user/user_login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"123456"}'

# 测试注册接口
curl -X POST http://localhost:8080/apps/basic/user/create_user \
  -H "Content-Type: application/json" \
  -d '{
    "name":"测试用户",
    "email":"test@example.com",
    "password":"123456",
    "phone":"13800138000",
    "role":4
  }'
```

### 6. 检查代理配置

前端的 API 请求会被代理到后端，检查配置：

**package.json**:
```json
{
  "proxy": "http://localhost:8080"
}
```

如果后端不是运行在 8080 端口，需要修改这个配置。

### 7. 临时禁用错误提示

如果 message 弹窗太多影响调试，可以临时注释：

**src/utils/request.js** 中的 `message.error(...)`

### 8. 检查后端响应头

后端需要设置正确的响应头：

```go
// 示例
w.Header().Set("Content-Type", "application/json")
w.Header().Set("Access-Control-Allow-Origin", "*")
w.Header().Set("Access-Control-Allow-Methods", "POST, GET, OPTIONS")
w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization, X-User-Email")
```

## 调试技巧

### 添加断点

在浏览器开发者工具的 Sources 标签中：
1. 找到 `src/pages/Login/index.jsx`
2. 在 `onLoginFinish` 函数的第一行添加断点
3. 点击登录按钮，程序会暂停
4. 逐步执行，观察变量值

### 使用 React DevTools

安装 React DevTools 浏览器插件，可以查看组件状态：
- Chrome: https://chrome.google.com/webstore/detail/react-developer-tools/
- Firefox: https://addons.mozilla.org/firefox/addon/react-devtools/

## 快速检查清单

- [ ] 后端服务已启动
- [ ] 后端监听在 localhost:8080
- [ ] 前端开发服务器已启动（localhost:3000）
- [ ] 浏览器控制台无 JavaScript 错误
- [ ] Network 标签显示请求已发送
- [ ] 后端返回 200 状态码
- [ ] 响应数据格式正确
- [ ] CORS 配置正确
