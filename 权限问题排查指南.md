# Admin 权限问题排查指南

## 问题描述

用户使用 admin 登录后，看不到"创建设备"和"新增/扣减库存"的入口。

## 可能的原因

### 1. 用户角色值不正确

**角色定义**（参考 `defines/constant/user_enum.go`）:
```go
UserRoleTypeSuperAdmin = 1  // 超级管理员
UserRoleTypeAdmin      = 2  // 管理员
UserRoleTypeTeacher    = 3  // 教师
UserRoleTypeStudent    = 4  // 学生
```

**前端权限判断**（参考 `frontend/src/store/authStore.js`）:
```javascript
get isAdmin() {
  return this.user && [1, 2, 3].includes(this.user.role);
}

get isTeacher() {
  return this.user && [1, 2, 3].includes(this.user.role);
}

get isSuperAdmin() {
  return this.user && this.user.role === 1;
}
```

**权限控制**:
- **新增设备**: 需要 `isAdmin` 权限（role 为 1, 2, 或 3）
- **库存操作**: 需要 `isAdmin` 权限（role 为 1, 2, 或 3）

### 2. 数据库中用户角色值检查

检查数据库中 admin 用户的 role 值:

```sql
SELECT user_id, email, name, role FROM user_tab WHERE email = 'admin@example.com';
```

确保 role 值为 1（超级管理员）、2（管理员）或 3（教师）。

### 步骤 3: 根据调试信息判断问题

#### 情况 A: role 值不正确

如果看到 `User role: 4`（学生角色），说明用户角色不对。

**解决方案**: 修改数据库中的用户角色

```sql
-- 将 admin@example.com 的角色改为超级管理员
UPDATE user_tab SET role = 1 WHERE email = 'admin@example.com';
```

然后重新登录。

#### 情况 B: role 值正确但 isAdmin 为 false

如果看到 `User role: 1` 但 `Is admin?: false`，说明前端权限判断有问题。

**解决方案 1**: 检查 localStorage 中的 userInfo

在浏览器控制台执行:
```javascript
JSON.parse(localStorage.getItem('userInfo'))
```

查看存储的用户信息中 `role` 字段的值和类型。

**解决方案 2**: 清除 localStorage 重新登录

在浏览器控制台执行:
```javascript
localStorage.clear();
```

然后刷新页面重新登录。

#### 情况 C: 登录接口返回的 user_info 中没有 role

如果登录响应中没有 `role` 字段，说明后端返回数据有问题。

**解决方案**: 检查后端代码

检查 `apps/basic/view/user_handler.go` 的 `UserLoginHandler` 方法，确保返回的 `UserInfo` 包含 `Role` 字段:

```go
userInfo := &pbbasic.LoginUserInfo{
    UserId: &user.UserID,
    Name:   &user.Name,
    Email:  &user.Email,
    Role:   &user.Role,  // 确保这一行存在
    Phone:  &user.Phone,
    Avatar: &user.Avatar,
}
```

## 快速修复方案

### 方案 1: 创建新的超级管理员用户

使用注册接口创建一个新用户，然后直接在数据库中修改其角色:

1. 前端注册页面注册一个新用户（role=4）
2. 在数据库中修改角色:
```sql
UPDATE user_tab SET role = 1 WHERE email = '你的邮箱@example.com';
```
3. 重新登录

### 方案 2: 直接在数据库中插入超级管理员

```sql
-- 生成密码的 MD5 + Salt
-- 密码: admin123
-- Salt: random_salt_12345

-- 插入超级管理员
INSERT INTO user_tab (
  user_id, email, name, phone, status, passwd, salt, role, ctime, mtime
) VALUES (
  'ADMIN_001',
  'superadmin@example.com',
  '超级管理员',
  '13800138000',
  1,
  'YOUR_HASHED_PASSWORD',  -- 需要用后端的加密方法生成
  'random_salt_12345',
  1,  -- 超级管理员
  UNIX_TIMESTAMP(),
  UNIX_TIMESTAMP()
);
```

**注意**: 密码需要使用后端的 `manager.HashPassword` 方法生成，不能直接使用明文。

### 方案 3: 使用后端提供的用户创建接口

调用 `/apps/basic/user/create_user` 接口创建用户，然后修改 role。

## 常见问题

### Q1: 为什么教师(role=3)也有管理权限？

A: 根据业务需求，教师需要管理设备和库存，并负责审批学生的借记申请，所以教师也被归类为"管理员"权限组。

### Q2: 如何临时提升某个学生的权限？

A: 执行 SQL:
```sql
UPDATE user_tab SET role = 3 WHERE email = '学生邮箱';
```
然后让该用户重新登录。

### Q3: 权限修改后为什么不生效？

A: 因为用户信息缓存在前端 localStorage 中。需要：
1. 清除浏览器 localStorage
2. 重新登录

或者在控制台执行:
```javascript
localStorage.clear();
location.reload();
```
