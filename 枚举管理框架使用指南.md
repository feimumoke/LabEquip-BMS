# 枚举管理框架使用指南

## 📖 概述

本系统实现了一个统一的枚举管理框架，前后端共享枚举定义，确保数据一致性。

### 特性

✅ **统一管理**：所有枚举值在后端集中定义和管理  
✅ **自动加载**：前端应用启动时自动加载并缓存所有枚举值  
✅ **类型安全**：提供完善的类型提示和验证  
✅ **便捷使用**：封装了常用的枚举操作方法和组件  
✅ **高性能**：一次加载，多次使用，避免重复请求

## 🔧 后端配置

### 1. 定义枚举

在 `defines/constant/` 目录下定义枚举：

```go
// equip_enum.go
package constant

type EquipCategory = int64

const (
    EquipCategoryGeneral     EquipCategory = 1 // 通用实验设备
    EquipCategoryChemical    EquipCategory = 2 // 化学实验设备
    EquipCategoryBiological  EquipCategory = 3 // 生物实验设备
    EquipCategoryPhysics     EquipCategory = 4 // 物理实验设备
    EquipCategoryElectronics EquipCategory = 5 // 电子实验设备
    EquipCategoryIT          EquipCategory = 6 // 计算机/信息设备
    EquipCategorySafety      EquipCategory = 7 // 安全与防护设备
    EquipCategorySpecial     EquipCategory = 8 // 特殊教学设备
)

// 导出给前端的枚举映射
var ExportCategoryNameToValue = map[string]interface{}{
    "通用实验设备":   EquipCategoryGeneral,
    "化学实验设备":   EquipCategoryChemical,
    "生物实验设备":   EquipCategoryBiological,
    "物理实验设备":   EquipCategoryPhysics,
    "电子实验设备":   EquipCategoryElectronics,
    "计算机/信息设备": EquipCategoryIT,
    "安全与防护设备":  EquipCategorySafety,
    "特殊教学设备":   EquipCategorySpecial,
}

// 注册枚举（在 init 函数中调用）
func init() {
    RegisterEnumValues("EquipCategory", ExportCategoryNameToValue)
}
```

## 💻 前端使用

### 1. 枚举自动加载

枚举值会在以下时机自动加载：

1. **应用启动时**（如果用户已登录）
2. **用户登录成功后**

无需手动调用加载方法。

### 2. 使用 enumStore

`enumStore` 提供了多种便捷的方法来操作枚举：

#### 方法列表

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getEnum(enumKey)` | enumKey: string | object | 获取枚举的原始数据 |
| `getEnumOptions(enumKey)` | enumKey: string | Array | 获取适用于 Select 的选项列表 |
| `getEnumLabel(enumKey, value)` | enumKey: string, value: number | string | 根据值获取显示名称 |
| `hasEnumValue(enumKey, value)` | enumKey: string, value: number | boolean | 检查枚举值是否存在 |

#### 使用示例

```javascript
import enumStore from '../../store/enumStore';

// 获取原始枚举数据
const equipCategories = enumStore.getEnum('EquipCategory');
// { "通用实验设备": 1, "化学实验设备": 2, ... }

// 获取选项列表（用于 Select）
const options = enumStore.getEnumOptions('EquipCategory');
// [{ label: "通用实验设备", value: 1 }, { label: "化学实验设备", value: 2 }, ...]

// 根据值获取显示名称
const label = enumStore.getEnumLabel('EquipCategory', 2);
// "化学实验设备"

// 检查值是否存在
const exists = enumStore.hasEnumValue('EquipCategory', 2);
// true
```

### 3. 使用 EnumSelect 组件

推荐使用封装好的 `EnumSelect` 组件：

```jsx
import EnumSelect from '../../components/EnumSelect';

// 在 Form 中使用
<Form.Item
  name="category_id"
  label="设备分类"
  rules={[{ required: true, message: '请选择设备分类' }]}
>
  <EnumSelect
    enumKey="EquipCategory"
    placeholder="请选择设备分类"
    size="large"
  />
</Form.Item>

// 作为受控组件使用
<EnumSelect
  enumKey="EquipCategory"
  value={categoryId}
  onChange={(value) => setCategoryId(value)}
  placeholder="请选择设备分类"
  showSearch={true}
  allowClear={true}
/>
```

#### EnumSelect 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| enumKey | string | - | 必填，枚举的key |
| placeholder | string | '请选择' | 占位符文本 |
| showSearch | boolean | true | 是否显示搜索框 |
| allowClear | boolean | false | 是否允许清除 |
| size | string | 'middle' | 尺寸: 'small' \| 'middle' \| 'large' |
| value | any | - | 选中的值 |
| onChange | function | - | 值变化回调 |

### 4. 在表格中显示枚举文本

```jsx
import { observer } from 'mobx-react-lite';
import enumStore from '../../store/enumStore';

const EquipmentTable = observer(() => {
  const columns = [
    {
      title: '设备分类',
      dataIndex: 'category_id',
      key: 'category_id',
      render: (value) => {
        const label = enumStore.getEnumLabel('EquipCategory', value);
        return <Tag color="purple">{label}</Tag>;
      },
    },
  ];

  return <Table columns={columns} dataSource={data} />;
});
```

### 5. 直接使用原生 Select 组件

如果需要自定义样式，也可以直接使用 Ant Design 的 Select：

```jsx
import { Select } from 'antd';
import { observer } from 'mobx-react-lite';
import enumStore from '../../store/enumStore';

const { Option } = Select;

const MyComponent = observer(() => {
  return (
    <Select placeholder="请选择设备分类">
      {enumStore.getEnumOptions('EquipCategory').map(option => (
        <Option key={option.value} value={option.value}>
          {option.label}
        </Option>
      ))}
    </Select>
  );
});
```

## 🎯 完整示例

### 示例1：创建设备页面

```jsx
import React, { useState } from 'react';
import { Form, Input, Modal, message } from 'antd';
import { observer } from 'mobx-react-lite';
import EnumSelect from '../../components/EnumSelect';
import { createEquip } from '../../api/equip';

const CreateEquipModal = observer(({ visible, onCancel, onSuccess }) => {
  const [form] = Form.useForm();
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (values) => {
    setLoading(true);
    try {
      await createEquip(values);
      message.success('创建成功');
      form.resetFields();
      onSuccess();
    } catch (error) {
      message.error('创建失败');
    } finally {
      setLoading(false);
    }
  };

  return (
    <Modal
      title="新增设备"
      visible={visible}
      onCancel={onCancel}
      onOk={() => form.submit()}
      confirmLoading={loading}
    >
      <Form form={form} onFinish={handleSubmit} layout="vertical">
        <Form.Item
          name="equip_name"
          label="设备名称"
          rules={[{ required: true, message: '请输入设备名称' }]}
        >
          <Input placeholder="请输入设备名称" />
        </Form.Item>

        <Form.Item
          name="category_id"
          label="设备分类"
          rules={[{ required: true, message: '请选择设备分类' }]}
        >
          <EnumSelect
            enumKey="EquipCategory"
            placeholder="请选择设备分类"
          />
        </Form.Item>

        <Form.Item name="description" label="描述">
          <Input.TextArea rows={4} placeholder="请输入设备描述" />
        </Form.Item>
      </Form>
    </Modal>
  );
});

export default CreateEquipModal;
```

### 示例2：设备列表展示

```jsx
import React, { useEffect, useState } from 'react';
import { Table, Tag } from 'antd';
import { observer } from 'mobx-react-lite';
import enumStore from '../../store/enumStore';
import { searchEquip } from '../../api/equip';

const EquipmentList = observer(() => {
  const [dataSource, setDataSource] = useState([]);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    loadData();
  }, []);

  const loadData = async () => {
    setLoading(true);
    try {
      const res = await searchEquip({});
      setDataSource(res.data?.list || []);
    } finally {
      setLoading(false);
    }
  };

  const columns = [
    {
      title: '设备名称',
      dataIndex: 'equip_name',
      key: 'equip_name',
    },
    {
      title: '设备分类',
      dataIndex: 'category_id',
      key: 'category_id',
      render: (value) => {
        const label = enumStore.getEnumLabel('EquipCategory', value);
        return <Tag color="purple">{label}</Tag>;
      },
    },
    {
      title: '描述',
      dataIndex: 'description',
      key: 'description',
    },
  ];

  return (
    <Table
      columns={columns}
      dataSource={dataSource}
      rowKey="equip_id"
      loading={loading}
    />
  );
});

export default EquipmentList;
```

## 📝 添加新枚举

### 步骤1：后端定义枚举

在 `defines/constant/` 下创建新的枚举文件，或在现有文件中添加：

```go
// my_enum.go
package constant

type MyStatusType = int64

const (
    MyStatusPending  MyStatusType = 1
    MyStatusApproved MyStatusType = 2
    MyStatusRejected MyStatusType = 3
)

var ExportMyStatusNameToValue = map[string]interface{}{
    "待审批": MyStatusPending,
    "已通过": MyStatusApproved,
    "已拒绝": MyStatusRejected,
}

func init() {
    RegisterEnumValues("MyStatus", ExportMyStatusNameToValue)
}
```

### 步骤2：前端使用

无需额外配置，直接使用即可：

```jsx
// 使用 EnumSelect 组件
<EnumSelect enumKey="MyStatus" />

// 或使用 enumStore
const options = enumStore.getEnumOptions('MyStatus');
const label = enumStore.getEnumLabel('MyStatus', 1); // "待审批"
```

### 常见问题

#### Q1: 枚举选项为空？

**原因**: 枚举可能还未加载完成

**解决方案**:
1. 检查控制台是否有 "✅ Enums loaded successfully" 日志
2. 确保用户已登录
3. 检查后端接口 `/api/apps/common/enums` 是否正常返回

#### Q2: 枚举显示的是数字而不是文本？

**原因**: 使用了错误的字段名

**解决方案**:
- 确保使用 `enumStore.getEnumLabel()` 而不是直接显示数值
- 检查枚举key是否正确

#### Q3: 新增的枚举前端获取不到？

**解决方案**:
1. 确保后端枚举已通过 `RegisterEnumValues` 注册
2. 清除浏览器缓存并重新登录
3. 检查后端接口返回的数据中是否包含新枚举

## 🎨 最佳实践

### 1. 使用 observer 包装组件

当使用 `enumStore` 时，务必使用 `observer` 包装组件以实现响应式：

```jsx
import { observer } from 'mobx-react-lite';

const MyComponent = observer(() => {
  // 使用 enumStore
  const options = enumStore.getEnumOptions('EquipCategory');
  // ...
});
```

### 2. 枚举命名规范

- **后端**: 使用 PascalCase，如 `EquipCategory`
- **前端**: 与后端保持一致
- **显示文本**: 使用中文，简洁明了

### 3. 表单验证

对枚举字段添加适当的验证规则：

```jsx
<Form.Item
  name="category_id"
  label="设备分类"
  rules={[
    { required: true, message: '请选择设备分类' },
    {
      validator: (_, value) => {
        if (value && !enumStore.hasEnumValue('EquipCategory', value)) {
          return Promise.reject(new Error('无效的设备分类'));
        }
        return Promise.resolve();
      },
    },
  ]}
>
  <EnumSelect enumKey="EquipCategory" />
</Form.Item>
```

### 4. 错误处理

在枚举未加载时提供友好的提示：

```jsx
const options = enumStore.isLoaded 
  ? enumStore.getEnumOptions('EquipCategory')
  : [];

if (!enumStore.isLoaded) {
  return <Spin tip="加载枚举中..." />;
}
```

## 📊 性能优化

### 1. 懒加载

枚举在首次需要时自动加载，无需预加载。

### 2. 缓存机制

枚举值加载后会缓存在 `enumStore` 中，避免重复请求。

### 3. 按需渲染

使用 MobX 的 `observer` 确保只在枚举数据变化时重新渲染。

## 📚 API 参考

### EnumStore

```typescript
class EnumStore {
  // 所有枚举值
  enums: Record<string, any>;
  
  // 是否已加载
  isLoaded: boolean;
  
  // 是否正在加载
  isLoading: boolean;
  
  // 加载所有枚举值
  loadEnums(): Promise<void>;
  
  // 获取枚举原始数据
  getEnum(enumKey: string): object;
  
  // 获取枚举选项列表
  getEnumOptions(enumKey: string): Array<{label: string, value: any}>;
  
  // 根据值获取显示名称
  getEnumLabel(enumKey: string, value: any): string;
  
  // 检查枚举值是否存在
  hasEnumValue(enumKey: string, value: any): boolean;
}
```

### EnumSelect Props

```typescript
interface EnumSelectProps {
  enumKey: string;           // 必填，枚举key
  placeholder?: string;      // 占位符，默认 '请选择'
  showSearch?: boolean;      // 是否显示搜索，默认 true
  allowClear?: boolean;      // 是否允许清除，默认 false
  size?: 'small' | 'middle' | 'large';  // 尺寸，默认 'middle'
  value?: any;              // 选中值
  onChange?: (value: any) => void;  // 值变化回调
  [key: string]: any;       // 其他 Select 组件支持的属性
}
```
