# 刷新页面枚举丢失问题修复

## 🐛 问题描述

用户反馈：**一刷新浏览器，前端枚举值就变成 int 值了，用户不知道啥意思**

### 问题示例

**登录后正常显示**：
```
状态：待审批
交易类型：增加库存
设备分类：化学实验设备
```

**刷新浏览器后变成数字**：
```
状态：1          ❌
交易类型：1      ❌
设备分类：2      ❌
```

---

## 🔍 问题分析

### 原因

1. **枚举数据存储在内存中**：
   - 枚举数据通过 MobX 的 `enumStore` 管理
   - 数据存储在 JavaScript 运行时内存中
   - 不是持久化存储（不在 LocalStorage 或 SessionStorage）

2. **刷新页面后内存清空**：
   - 用户刷新浏览器（F5 或 Ctrl+R）
   - JavaScript 运行时重新启动
   - MobX store 中的数据全部丢失
   - `enumStore.enums` 变成空对象 `{}`

3. **之前的加载逻辑问题**：
   ```javascript
   // App.js 中被注释掉了
   // useEffect(() => {
   //   if (isAuthenticated()) {
   //     enumStore.loadEnums();
   //   }
   // }, []);
   ```
   - 应用启动时不会自动加载枚举
   - 只有登录成功后才加载
   - 刷新页面后不会重新加载

4. **枚举未加载时的显示逻辑**：
   ```javascript
   // enumStore.getEnumLabel() 的实现
   getEnumLabel(enumKey, value) {
     const enumData = this.getEnum(enumKey);
     
     // 如果枚举数据为空
     if (!enumData || Object.keys(enumData).length === 0) {
       return String(value);  // ❌ 直接返回数字
     }
     
     // 查找对应的文本
     for (const [label, val] of Object.entries(enumData)) {
       if (val === value) {
         return label;
       }
     }
     
     return String(value);  // ❌ 未找到时返回数字
   }
   ```

## ✅ 解决方案

### 方案 1：应用启动时自动加载枚举（已实现）

**文件**：`frontend/src/App.js`

**修改前** ❌：
```javascript
function App() {
  // 注释掉应用启动时的枚举加载
  // 枚举只在登录成功后加载
  // useEffect(() => {
  //   if (isAuthenticated()) {
  //     enumStore.loadEnums();
  //   }
  // }, []);

  return (
    // ...
  );
}
```

**修改后** ✅：
```javascript
function App() {
  // 应用启动时检查登录状态，如果已登录则加载枚举
  useEffect(() => {
    if (isAuthenticated()) {
      console.log('🔄 User is authenticated, loading enums...');
      enumStore.loadEnums();
    }
  }, []);

  return (
    // ...
  );
}
```

**工作原理**：
1. 应用启动时，`useEffect` 执行
2. 检查用户是否已登录（`isAuthenticated()` 检查 LocalStorage 中的 token）
3. 如果已登录，自动加载枚举
4. 枚举加载完成后，所有页面都可以正常显示

### 方案 2：枚举持久化（可选的进一步优化）

如果想避免每次刷新都请求后端，可以将枚举数据持久化到 LocalStorage。

**文件**：`frontend/src/store/enumStore.js`

```javascript
import { makeAutoObservable } from 'mobx';
import { getEnums } from '../api/enum';

const ENUM_CACHE_KEY = 'enum_cache';
const ENUM_CACHE_TIMESTAMP_KEY = 'enum_cache_timestamp';
const CACHE_DURATION = 24 * 60 * 60 * 1000; // 24小时

class EnumStore {
  enums = {};
  isLoaded = false;
  isLoading = false;

  constructor() {
    makeAutoObservable(this);
    // 应用启动时尝试从缓存加载
    this.loadFromCache();
  }

  // 从 LocalStorage 加载缓存
  loadFromCache() {
    try {
      const cachedData = localStorage.getItem(ENUM_CACHE_KEY);
      const timestamp = localStorage.getItem(ENUM_CACHE_TIMESTAMP_KEY);
      
      if (cachedData && timestamp) {
        const age = Date.now() - parseInt(timestamp);
        
        // 如果缓存未过期
        if (age < CACHE_DURATION) {
          this.enums = JSON.parse(cachedData);
          this.isLoaded = true;
          console.log('✅ Enums loaded from cache');
          return true;
        } else {
          console.log('⏰ Enum cache expired, will reload from server');
        }
      }
    } catch (error) {
      console.error('❌ Failed to load enums from cache:', error);
    }
    return false;
  }

  // 保存到 LocalStorage
  saveToCache() {
    try {
      localStorage.setItem(ENUM_CACHE_KEY, JSON.stringify(this.enums));
      localStorage.setItem(ENUM_CACHE_TIMESTAMP_KEY, String(Date.now()));
      console.log('💾 Enums saved to cache');
    } catch (error) {
      console.error('❌ Failed to save enums to cache:', error);
    }
  }

  async loadEnums() {
    if (this.isLoaded || this.isLoading) {
      console.log('⏭️ Enums already loaded or loading, skipping...');
      return;
    }

    this.isLoading = true;
    try {
      console.log('🔄 Loading enums from server...');
      const res = await getEnums();
      if (res && res.retcode === 0 && res.data) {
        this.enums = res.data;
        this.isLoaded = true;
        
        // 保存到缓存
        this.saveToCache();
        
        console.log('✅ Enums loaded successfully:', this.enums);
        console.log('📋 Available enum keys:', Object.keys(this.enums));
        
        if (this.enums.BorrowTaskStatus) {
          console.log('📊 BorrowTaskStatus enum:', this.enums.BorrowTaskStatus);
        }
      }
    } catch (error) {
      console.error('❌ Failed to load enums:', error);
    } finally {
      this.isLoading = false;
    }
  }

  // 其他方法保持不变...
}

export default new EnumStore();
```

**优点**：
- ✅ 刷新页面时从缓存加载，速度快
- ✅ 减少后端请求
- ✅ 24小时后自动过期，重新从服务器加载

**缺点**：
- ⚠️ 如果后端枚举更新，需要清除缓存或等待过期

## 🎉 总结

### 问题

- ❌ 刷新页面后，枚举值变成数字（如 1、2、3）
- ❌ 用户不知道数字的含义

### 原因

- 枚举数据存储在内存中（MobX store）
- 刷新页面后内存清空
- 应用启动时不会自动重新加载枚举

### 解决方案

- ✅ 在 App.js 中恢复应用启动时的枚举加载逻辑
- ✅ 检查用户是否已登录，如果已登录则自动加载枚举
- ✅ 防止重复加载（enumStore 内部有检查）

### 效果

- ✅ 刷新页面后枚举仍然显示为文本
- ✅ 用户体验改善，不会看到数字
- ✅ 无需额外操作，自动恢复

### 后续优化（可选）

- 💡 将枚举缓存到 LocalStorage（24小时有效）
- 💡 减少后端请求次数
- 💡 加快页面加载速度
