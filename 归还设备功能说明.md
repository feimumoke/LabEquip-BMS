# 归还设备功能说明

## 🎯 功能概述

在"我的借记"页面，学生可以对**进行中状态（状态4）**的借记任务执行"归还设备"操作，表示将借用的设备归还到实验室。

---

## 📋 状态流转

### 借记任务完整流程（含归还）

```
1. 待审批 (Pending, 1)
   └─ 学生创建借记任务
   
   ↓ (教师审批通过)
   
2. 已审批 (Approval, 2)
   └─ 审批通过，系统分配库存
   
   ↓ (库存分配成功)
   
3. 已分配 (Allocate, 3)
   └─ 学生可以拿走设备
   
   ↓ (学生拿走设备)
   
4. 进行中 (Ongoing, 4) ← 【可以执行"归还设备"操作】
   ├─ 学生已拿走设备
   ├─ 设备在学生手中
   └─ ✅ 显示"归还设备"按钮
   
   ↓ (学生归还设备)
   
8. 已归还 (Done, 8)
   └─ 任务完成 ✅
```

## 🔧 实现细节

### 1. 后端接口

**接口路径**：`/apps/bms/borrow/return_borrow`

**请求参数**：
```json
{
  "borrow_id": "借记任务ID",
  "return_qty": 3,  // 归还数量（必填）
  "code_list": ["设备编码1", "设备编码2"]  // 可选，传空数组
}
```

**响应**：
```json
{
  "retcode": 0,
  "message": "success",
  "data": {}
}
```

**后端逻辑**：
1. 验证借记任务状态为"进行中"（状态4）
2. 验证归还数量不超过借用数量
3. 将借记任务状态更新为"已归还"（状态8）
4. 更新库存状态：
   - `borrowed_qty` 减少（借出数量）
   - `on_hand_qty` 增加（在手数量）
   - `avail_qty` 增加（可用数量）
5. 记录操作日志
6. 记录三级账

#### 页面组件

**文件**：`frontend/src/pages/Borrow/MyTasks.jsx`

**关键代码**：

```javascript
import { RollbackOutlined } from '@ant-design/icons';
import { returnBorrow } from '../../api/borrow';

// 归还设备的处理函数
const handleReturn = async (record) => {
  Modal.confirm({
    title: '🔄 确认归还设备',
    content: (
      <div style={{ padding: '16px 0' }}>
        <div style={{ 
          background: '#f6ffed', 
          border: '1px solid #b7eb8f',
          borderRadius: 8,
          padding: 16,
          marginBottom: 16
        }}>
          <p style={{ margin: 0, color: '#52c41a', fontWeight: 600 }}>
            确认归还设备后，状态将变更为"已归还"，任务完成
          </p>
        </div>
        <p>任务ID：{record.task_id}</p>
        <p>设备：{record.equip_name || record.equip_id}</p>
        <p>实验室：{record.lab_name || record.lab_code}</p>
        <p>归还数量：{record.borrow_qty}</p>
        <div style={{ marginTop: 16, padding: 12, background: '#e6f7ff' }}>
          <p>💡 提示：请确认设备完好无损后再归还</p>
        </div>
      </div>
    ),
    okText: '确认归还',
    cancelText: '取消',
    okButtonProps: { type: 'primary' },
    onOk: async () => {
      try {
        await returnBorrow({ 
          borrow_id: record.task_id,
          return_qty: record.borrow_qty,
          code_list: []
        });
        message.success('✅ 归还成功，感谢您的使用');
        loadData();
      } catch (error) {
        message.error('❌ 归还失败');
      }
    },
  });
};

// 表格列定义中的操作列
{
  title: '操作',
  key: 'action',
  width: 150,
  fixed: 'right',
  render: (_, record) => (
    <Space>
      {record.task_status === 1 && ( // 待审批状态可以取消
        <Button danger onClick={() => handleCancel(record)}>
          取消
        </Button>
      )}
      {record.task_status === 3 && ( // 已分配状态可以拿走设备
        <Button type="primary" onClick={() => handleTakeBorrow(record)}>
          拿走设备
        </Button>
      )}
      {record.task_status === 4 && ( // 进行中状态可以归还
        <Button 
          type="primary" 
          icon={<RollbackOutlined />}
          onClick={() => handleReturn(record)}
          style={{ background: '#52c41a', borderColor: '#52c41a' }}
        >
          归还设备
        </Button>
      )}
    </Space>
  ),
}
```

## 💡 使用提示

### 给学生的提示

1. **归还时机**：使用完设备后应及时归还
2. **设备检查**：归还前请检查设备是否完好
3. **归还确认**：归还后任务状态变为"已归还"，无法撤销
4. **状态查看**：可以在列表中查看已归还的任务

### 给教师的提示

1. **验收设备**：学生归还时应检查设备状态
2. **问题处理**：如发现设备损坏，及时记录
3. **库存核对**：定期核对实际库存与系统记录

## 📈 后续优化建议

### 1. 部分归还

支持学生分批次归还设备：
- 借了 5 个，可以先归还 3 个
- 再归还剩余 2 个
- 记录每次归还的数量和时间

### 2. 设备状态评估

归还时评估设备状态：
- 完好
- 轻微损坏
- 严重损坏
- 需要维修

### 3. 拍照记录

- 归还时拍照记录设备状态
- 与拿走时的照片对比
- 作为损坏责任的依据

### 4. 自动提醒

- 超期未归还自动提醒
- 即将到期提醒
- 归还成功通知教师
