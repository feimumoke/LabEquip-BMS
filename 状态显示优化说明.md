# 状态显示优化说明

## 📋 问题描述

用户反馈：**状态不要显示数值，要显示枚举对应的key**

### 问题示例

❌ **错误显示**：显示数字 `1`, `2`, `3` 等
✅ **正确显示**：显示文本 `待审批`, `已审批`, `已分配` 等

---

## 🔍 问题分析

### 1. 当前实现

所有借记相关页面已经使用了 `enumStore.getEnumLabel('BorrowTaskStatus', status)` 来显示状态：

#### 我的借记页面 (`MyTasks.jsx`)

```javascript
{
  title: '状态',
  dataIndex: 'task_status',
  key: 'task_status',
  render: (status) => {
    const label = enumStore.getEnumLabel('BorrowTaskStatus', status);
    const color = getStatusColor(status);
    return <Tag color={color}>{label}</Tag>;
  },
}
```

#### 审批管理页面 (`Approve.jsx`)

```javascript
{
  title: '状态',
  dataIndex: 'task_status',
  key: 'task_status',
  render: (status) => {
    const label = enumStore.getEnumLabel('BorrowTaskStatus', status);
    return <Tag color="orange">{label}</Tag>;
  },
}
```

#### 所有借记页面 (`AllTasks.jsx`)

```javascript
{
  title: '状态',
  dataIndex: 'task_status',
  key: 'task_status',
  render: (status) => {
    const label = enumStore.getEnumLabel('BorrowTaskStatus', status);
    const color = getStatusColor(status);
    return <Tag color={color}>{label}</Tag>;
  },
}
```

### 2. 可能的原因

如果状态显示的是数字而不是文本，可能的原因有：

#### 原因 1：枚举未加载

- 用户在登录后没有正确加载枚举
- `enumStore.isLoaded` 为 `false`
- `enumStore.enums` 为空对象 `{}`

#### 原因 2：枚举 Key 不匹配

- 后端注册的枚举 key 与前端请求的不一致
- 后端：`BorrowTaskStatus`
- 前端：可能拼写错误或大小写不匹配

#### 原因 3：枚举值类型不匹配

- 后端返回的值类型与前端比较的类型不一致
- 后端：`int64` (如 `1`, `2`, `3`)
- 前端：可能是字符串 (如 `"1"`, `"2"`, `"3"`)

## ✅ 解决方案

### 1. 增强 enumStore 调试信息

已更新 `frontend/src/store/enumStore.js`，添加了详细的日志输出：

```javascript
async loadEnums() {
  // ... 省略部分代码 ...
  
  console.log('✅ Enums loaded successfully:', this.enums);
  console.log('📋 Available enum keys:', Object.keys(this.enums));
  
  // 特别输出 BorrowTaskStatus 枚举用于调试
  if (this.enums.BorrowTaskStatus) {
    console.log('📊 BorrowTaskStatus enum:', this.enums.BorrowTaskStatus);
  } else {
    console.warn('⚠️ BorrowTaskStatus not found in enums!');
  }
}

getEnumLabel(enumKey, value) {
  const enumData = this.getEnum(enumKey);
  
  // 如果枚举数据为空，返回原值
  if (!enumData || Object.keys(enumData).length === 0) {
    console.warn(`⚠️ Enum '${enumKey}' is empty, returning value: ${value}`);
    return String(value);
  }
  
  // ... 查找逻辑 ...
  
  console.warn(`⚠️ Value ${value} not found in enum '${enumKey}'. Available:`, enumData);
  return String(value);
}
```

### 2. 确保枚举在登录后加载

在 `frontend/src/pages/Login/index.jsx` 中，登录成功后立即加载枚举：

```javascript
const handleLogin = async (values) => {
  try {
    const res = await login({
      email: values.email,
      password: values.password,
      client_type: 1,
    });

    if (res.retcode === 0) {
      // 保存登录信息
      authStore.setUser(res.data.user_info);
      setToken(res.data.token);
      setUserEmail(res.data.user_info.email);
      setUserInfo(JSON.stringify(res.data.user_info));

      // ✅ 登录成功后立即加载枚举
      await enumStore.loadEnums();

      message.success('登录成功');
      navigate('/inventory/list');
    }
  } catch (error) {
    message.error('登录失败');
  }
};
```

## 🔧 常见问题修复

### 问题 1：枚举未加载

**症状**: 所有状态都显示数字

**原因**: `enumStore.loadEnums()` 没有被调用或失败

**解决方案**:

1. 确保登录成功后调用了 `enumStore.loadEnums()`
2. 检查 `/api/apps/common/enums` 接口是否正常
3. 检查浏览器控制台是否有错误

### 问题 2：枚举 Key 不存在

**症状**: 特定枚举的状态显示数字，其他枚举正常

**原因**: 后端没有注册该枚举

**解决方案**:

1. 检查后端 `defines/constant/borrow_enum.go` 中的 `init()` 函数
2. 确认 `RegisterEnumValues("BorrowTaskStatus", ...)` 被调用
3. 重启后端服务

### 问题 3：值类型不匹配

**症状**: 枚举已加载，但状态仍显示数字

**原因**: 后端返回的值类型与前端比较的类型不一致

**解决方案**:

检查后端返回的数据类型：

```go
// 确保返回的是 int64，而不是字符串
var ExportBorrowTaskStatusNameToValue = map[string]interface{}{
	"待审批": BorrowTaskStatusPending,  // int64(1)，不是 "1"
	...
}
```

### 问题 4：枚举加载时机过早

**症状**: 第一次访问页面时状态显示数字，刷新后正常

**原因**: 枚举在登录前加载，导致没有 token 无法访问接口

**解决方案**:

确保枚举在登录后加载，而不是在 `App.js` 的 `useEffect` 中加载。
