# 上传接口修复总结

## 🐛 问题

用户反馈：**`/apps/common/upload_file` 上传时没带上上传文件 `file_name`，导致文件没有后缀，上传之后返回的 URL 没有累加**

---

## ✅ 已修复

### 1. 后端修复

**文件**：`apps/common/view/common_view.go`

**修复内容**：
- ✅ 从 `req.GetFileName()` 获取原始文件名
- ✅ 提取文件扩展名（如 `.jpg`、`.png`）
- ✅ 生成的文件名包含扩展名：`{UUID}.{ext}`
- ✅ 返回完整的可访问 URL：`/uploads/{date}/{uuid}.{ext}`

**修复前** ❌：
```go
// 文件名：uuid（无扩展名）
// 返回：./upload/20240113/uuid
generateUUID, _ := uuid.NewUUID()
path += generateUUID.String()
return &pbbasic.UploadFileResponse{
	Url: convert.String(path),
}
```

**修复后** ✅：
```go
// 获取扩展名
ext := ""
if idx := strings.LastIndex(originalFilename, "."); idx != -1 {
	ext = originalFilename[idx:]  // 如 ".jpg"
} else {
	ext = ".jpg"  // 默认扩展名
}

// 生成文件名：uuid.jpg
generateUUID, _ := uuid.NewUUID()
filename := generateUUID.String() + ext
fullPath := uploadDir + filename

// 返回 URL：/uploads/20240113/uuid.jpg
fileUrl := "/uploads/" + dateStr + "/" + filename
return &pbbasic.UploadFileResponse{
	Url: convert.String(fileUrl),
}
```

### 2. 前端修复

**文件**：`frontend/src/components/ImageUpload.jsx`

**修复内容**：
- ✅ 修改 URL 字段映射：`file.response?.url`（原来是 `file.response?.file_url`）

**修复前** ❌：
```javascript
const urls = newFileList
  .filter((file) => file.status === 'done')
  .map((file) => file.response?.file_url || file.url);  // ❌ file_url 字段不存在
```

**修复后** ✅：
```javascript
const urls = newFileList
  .filter((file) => file.status === 'done')
  .map((file) => file.response?.url || file.url);  // ✅ 使用 url 字段
```

## 🔄 完整流程

### 上传流程

```
1. 前端选择图片文件（如 photo.jpg）
   ↓
2. FormData 包含：
   - file: [binary data]
   - file.name: "photo.jpg"
   ↓
3. 后端接收：
   - req.GetFileName() = "photo.jpg"
   - req.File = [binary data]
   ↓
4. 后端处理：
   - 提取扩展名：".jpg"
   - 生成文件名："{uuid}.jpg"
   - 保存到：./uploads/20240113/{uuid}.jpg
   - 返回 URL：/uploads/20240113/{uuid}.jpg
   ↓
5. 前端接收：
   - response.data.url = "/uploads/20240113/{uuid}.jpg"
   - 保存到 fileList
   - 显示缩略图 ✅
```

## ⚠️ 注意事项

### 1. 静态文件服务

需要配置 Gin 或 Nginx 提供 `/uploads/` 目录的静态文件服务：

**Gin 配置**（`server/api/main.go`）：
```go
router.Static("/uploads", "./uploads")
```

**Nginx 配置**：
```nginx
location /uploads/ {
    alias /path/to/LabEquip-BMS/uploads/;
    expires 30d;
    access_log off;
}
```

---

### 2. 目录权限

```bash
# 确保上传目录存在且有写权限
mkdir -p ./uploads
chmod 755 ./uploads
```

---

### 3. Protobuf 定义

**文件**：`api_idl/apps/basic/upload_file.proto`

```protobuf
message UploadFileRequest {
  required string file_name = 1;  // 文件名（包含扩展名）
  required bytes file = 2;        // 文件内容
}

message UploadFileResponse {
  optional string url = 1;  // 文件访问 URL
}
```

**注意**：
- `UploadFileResponse` 只有 `url` 字段，没有 `file_name` 字段
- 前端需要使用 `response.data.url` 获取 URL
