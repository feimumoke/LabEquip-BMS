## 1. 数据库概述

### 1.1 系统简介

实验设备借记管理系统（LabEquip-BMS）是一个用于管理实验室设备借记的系统，包含用户管理、设备管理、库存管理、借记管理等核心功能。

### 1.2 表结构总览

系统共包含 **11 张核心表**，分为以下几类：

| 分类 | 表名 | 说明 |
|------|------|------|
| **用户相关** | user_tab | 用户基本信息表 |
| | user_login_tab | 用户登录记录表 |
| | login_session_tab | 登录Session表 |
| **基础信息** | lab_tab | 实验室信息表 |
| | equip_tab | 设备信息表 |
| **库存管理** | inventory_tab | 库存表 |
| | inventory_task_tab | 库存任务表 |
| | inventory_task_log_tab | 库存任务日志表 |
| **借记管理** | borrow_task_tab | 借记任务表 |
| | borrow_task_log_tab | 借记任务日志表 |
| **交易记录** | transaction_log_tab | 交易日志表(三级账) |

### 2.3 库存管理表

#### 2.3.1 inventory_tab (库存表)

**用途**: 记录设备在各个实验室的库存情况

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | BIGINT | 主键 | 自增 |
| lab_id | VARCHAR(64) | 实验室ID | |
| equip_id | VARCHAR(64) | 设备ID | |
| total_qty | BIGINT | 总库存数量 | |
| on_hand_qty | BIGINT | 在手库存 | = total - borrowed |
| available_qty | BIGINT | 可用库存 | = total - borrowed - allocated |
| borrowed_qty | BIGINT | 已借出数量 | 已拿走 |
| allocated_qty | BIGINT | 已分配数量 | 未拿走 |
| operator | VARCHAR(64) | 操作人 | |
| ctime | BIGINT | 创建时间 | Unix时间戳 |
| mtime | BIGINT | 修改时间 | Unix时间戳 |

**索引**:
- 主键: `id`
- 唯一索引: `(lab_id, equip_id)` - 一个实验室的一种设备只有一条库存记录
- 普通索引: `equip_id`, `available_qty`

**库存关系**:
```
总库存 (total_qty) = 在手库存 (on_hand_qty) + 已借出 (borrowed_qty)
在手库存 (on_hand_qty) = 可用库存 (available_qty) + 已分配 (allocated_qty)
```

**状态流转**:
```
创建借记 → allocated_qty 增加, available_qty 减少
批准拿走 → allocated_qty 减少, borrowed_qty 增加
归还设备 → borrowed_qty 减少, on_hand_qty 增加, available_qty 增加
```

### 2.4 借记管理表

#### 2.4.1 borrow_task_tab (借记任务表)

**用途**: 记录设备借记任务

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | BIGINT | 主键 | 自增 |
| task_id | VARCHAR(64) | 任务ID | 唯一 |
| equip_id | VARCHAR(64) | 设备ID | |
| lab_id | VARCHAR(64) | 实验室ID | |
| borrow_qty | BIGINT | 借记数量 | |
| task_status | BIGINT | 任务状态 | 见枚举值 |
| creator | VARCHAR(64) | 创建人 | 借记人 |
| approval | VARCHAR(64) | 审批人 | 教师 |
| ctime | BIGINT | 创建时间 | Unix时间戳 |
| mtime | BIGINT | 修改时间 | Unix时间戳 |

**索引**:
- 主键: `id`
- 唯一索引: `task_id`
- 普通索引: `(equip_id, lab_id)`, `task_status`, `creator`, `approval`

**状态流转**:
```
1. Pending (待分配) - 创建借记，库存不足
   ↓
2. Allocate (已分配) - 分配库存成功
   ↓
3. Approval (已审批) - 教师审批通过
   ↓
4. Ongoing (进行中) - 学生已拿走设备
   ↓
8. Done (已归还) - 归还设备

取消流程:
1/2 → 9. Cancel (已取消) - 学生取消
3 → 7. Reject (已拒绝) - 教师拒绝
```

## 5. 业务流程

### 5.1 借记流程

```
1. 学生创建借记任务
   → borrow_task_tab (status = 1 Pending)
   → 尝试分配库存

2a. 库存充足
   → inventory_tab (available_qty↓, allocated_qty↑)
   → transaction_log_tab (trans_type = 3 Allocate)
   → borrow_task_tab (status = 2 Allocate)

2b. 库存不足
   → borrow_task_tab (status = 1 Pending)
   → 等待库存补充或任务取消

3. 教师审批
   3a. 审批通过
      → borrow_task_tab (status = 3 Approval)
      → borrow_task_log_tab (记录审批)
   
   3b. 审批拒绝
      → borrow_task_tab (status = 7 Reject)
      → inventory_tab (allocated_qty↓, available_qty↑)
      → transaction_log_tab (trans_type = 6 Reject)

4. 学生拿走设备
   → borrow_task_tab (status = 4 Ongoing)
   → inventory_tab (allocated_qty↓, borrowed_qty↑)
   → transaction_log_tab (trans_type = 4 Borrow)

5. 归还设备
   → borrow_task_tab (status = 8 Done)
   → inventory_tab (borrowed_qty↓, on_hand_qty↑, available_qty↑)
   → transaction_log_tab (trans_type = 5 Return)
```

### 5.2 库存管理流程

```
1. 增加库存
   → inventory_task_tab (task_type = 1 Increase)
   → inventory_tab (total_qty↑, on_hand_qty↑, available_qty↑)
   → transaction_log_tab (trans_type = 1 Increase)
   → inventory_task_tab (task_status = 2 Done)

2. 减少库存
   → inventory_task_tab (task_type = 2 Decrease)
   → inventory_tab (total_qty↓, on_hand_qty↓, available_qty↓)
   → transaction_log_tab (trans_type = 2 Decrease)
   → inventory_task_tab (task_status = 2 Done)
```

## 6. 性能优化建议

### 6.1 分表策略

**transaction_log_tab (三级账表)**:
- 按月分表: `transaction_log_tab_202401`, `transaction_log_tab_202402`
- 历史数据归档: 超过1年的数据迁移到历史表

### 6.2 查询优化

1. **高频查询使用复合索引**
   ```sql
   -- 查询某实验室某设备的库存
   SELECT * FROM inventory_tab WHERE lab_id = ? AND equip_id = ?;
   -- 使用复合索引: (lab_id, equip_id)
   ```

2. **避免全表扫描**
   ```sql
   -- 不推荐
   SELECT * FROM transaction_log_tab WHERE ctime > ?;
   
   -- 推荐: 加上其他条件缩小范围
   SELECT * FROM transaction_log_tab 
   WHERE lab_id = ? AND ctime > ?;
   ```

3. **使用EXPLAIN分析查询**
   ```sql
   EXPLAIN SELECT * FROM borrow_task_tab 
   WHERE task_status = 2 AND creator = ?;
   ```

### 6.3 数据归档

建议定期归档以下表的历史数据:
- `user_login_tab` - 保留最近6个月
- `transaction_log_tab` - 保留最近1年
- `borrow_task_log_tab` - 保留最近1年
- `inventory_task_log_tab` - 保留最近1年

## 7. 安全建议

### 7.1 密码存储

- 使用MD5 + 随机盐值
- 盐值存储在 `user_tab.salt`
- 密码不可逆加密

### 7.2 Session管理

- Session有效期7天
- 定期清理过期Session
- 使用Redis缓存Session

### 7.3 权限控制

- 基于角色的访问控制(RBAC)
- API层拦截器验证权限
- 敏感操作记录日志
