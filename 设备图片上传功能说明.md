# 设备图片上传功能说明

## 📋 功能概述

为设备管理模块增加了图片上传功能，支持每个设备上传最多 10 张图片，并提供统一的文件上传接口。

---

## 🎯 功能特性

### 1. 统一文件上传接口

**接口路径**：`/apps/common/upload_file`

**功能**：
- ✅ 支持图片上传（JPG、JPEG、PNG、GIF、WebP）
- ✅ 单文件大小限制：10MB
- ✅ 自动生成唯一文件名（MD5 + 时间戳）
- ✅ 返回文件访问 URL

**实现文件**：
- `apps/common/view/upload_file_handler.go` - 上传处理器
- `apps/common/view/common_router.go` - 路由注册

**请求示例**：
```bash
POST /apps/common/upload_file
Content-Type: multipart/form-data

file: [binary data]
```

**响应示例**：
```json
{
  "retcode": 0,
  "message": "",
  "data": {
    "file_url": "/uploads/a1b2c3d4e5f6g7h8_1705123456.jpg",
    "file_name": "equipment_photo.jpg"
  }
}
```

---

### 2. 设备修改接口

**接口路径**：`/apps/basic/equip/update_equip`

**功能**：
- ✅ 支持修改设备基本信息
- ✅ 支持更新设备图片列表
- ✅ 图片数量限制：最多 10 张

**实现文件**：
- `api_idl/apps/basic/update_equip.proto` - Protobuf 定义
- `apps/basic/view/equip_handler.go` - 更新处理器
- `apps/basic/manager/equip_manager.go` - 数据库操作

**请求示例**：
```json
{
  "equip_id": "EQ20240101001",
  "equip_name": "高精度电子天平",
  "category_id": 1,
  "model": "FA2004",
  "description": "精度 0.1mg，最大称量 200g",
  "image_list": [
    "/uploads/a1b2c3d4e5f6g7h8_1705123456.jpg",
    "/uploads/b2c3d4e5f6g7h8i9_1705123457.jpg"
  ]
}
```

**响应示例**：
```json
{
  "retcode": 0,
  "message": "",
  "data": {
    "success": true
  }
}
```

### 3. 数据库结构

**表名**：`equip_tab`

**新增字段**：
```sql
`images` TEXT COMMENT '设备图片URL列表(JSON格式,最多10张)'
```

**存储格式**：JSON 字符串数组
```json
["http://example.com/uploads/image1.jpg", "http://example.com/uploads/image2.jpg"]
```

**SQL 更新语句**：
```sql
ALTER TABLE `equip_tab` 
ADD COLUMN `images` TEXT COMMENT '设备图片URL列表(JSON格式,最多10张)' 
AFTER `model`;
```

## 🔄 完整流程

### 创建设备流程

```
1. 用户点击"新增设备"
   ↓
2. 填写设备基本信息
   ↓
3. 上传设备图片（可选，最多10张）
   - 选择图片文件
   - 自动调用 /apps/common/upload_file 上传
   - 返回图片 URL
   ↓
4. 点击"创建"
   ↓
5. 调用 /apps/basic/equip/create_equip
   - 保存设备信息
   - 保存图片 URL 列表（JSON 格式）
   ↓
6. 创建成功，刷新列表
```

---

### 编辑设备流程

```
1. 用户点击"编辑"按钮
   ↓
2. 打开编辑表单，回显设备信息
   - 显示已上传的图片
   ↓
3. 修改设备信息 / 添加或删除图片
   - 新上传的图片调用 /apps/common/upload_file
   - 删除图片直接从列表移除
   ↓
4. 点击"更新"
   ↓
5. 调用 /apps/basic/equip/update_equip
   - 更新设备信息
   - 更新图片 URL 列表
   ↓
6. 更新成功，刷新列表
```

---

### 查看图片流程

```
1. 用户在设备列表看到图片数量（如"3张"）
   ↓
2. 点击图片数量
   ↓
3. 打开图片预览弹窗
   - 显示所有图片缩略图
   ↓
4. 点击任意图片
   ↓
5. 打开大图预览
   - 支持左右切换
   - 支持缩放
```

## 📁 文件存储

### 存储目录

**后端存储路径**：`./uploads/`

**访问路径**：`/uploads/{filename}`

**文件命名规则**：`{MD5前16位}_{时间戳}.{扩展名}`

**示例**：
- 原文件名：`equipment_photo.jpg`
- 存储文件名：`a1b2c3d4e5f6g7h8_1705123456.jpg`
- 访问 URL：`/uploads/a1b2c3d4e5f6g7h8_1705123456.jpg`

### 静态文件服务

需要在 Nginx 或 Gin 中配置静态文件服务：

**Gin 配置示例**：
```go
// server/api/main.go
router.Static("/uploads", "./uploads")
```

**Nginx 配置示例**：
```nginx
location /uploads/ {
    alias /path/to/LabEquip-BMS/uploads/;
    expires 30d;
    access_log off;
}
```

## ⚠️ 注意事项

### 1. 文件大小限制
- 单文件最大 10MB
- 前端和后端都有校验
- 超过限制会提示错误

### 2. 文件类型限制
- 仅支持图片格式：JPG、JPEG、PNG、GIF、WebP
- 其他格式会被拒绝

### 3. 数量限制
- 每个设备最多 10 张图片
- 前端组件会自动隐藏上传按钮
- 后端会校验数量

### 4. 存储空间
- 需要定期清理未使用的图片
- 建议实现图片引用计数机制
- 删除设备时考虑是否删除图片文件

### 5. 安全性
- 文件名使用 MD5 + 时间戳，防止重名
- 需要配置 CORS 允许跨域上传
- 建议添加文件内容校验（防止恶意文件）
