# 我的借记页面显示修复

## 🐛 问题描述

用户反馈：
1. ❌ 我的借记页面没有展示实验室名称
2. ❌ 设备列没有展示设备名称

---

## 🔍 问题分析

### 后端返回的数据结构

根据 proto 定义 (`search_borrow_task.proto`)，`BorrowTaskInfo` 包含以下字段：

```protobuf
message BorrowTaskInfo {
  required string task_id = 11;
  required string equip_id = 1;
  required string equip_name = 2;        // ✅ 设备名称
  optional string lab_code = 3;
  optional string lab_name = 4;          // ✅ 实验室名称
  optional int64 borrow_qty = 5;
  optional int64 task_status = 6;
  required string operator = 7;
  optional int64 ctime = 9;
  repeated BorrowTaskLog log_list = 10;
}
```

**后端已经返回了**：
- ✅ `equip_name` - 设备名称
- ✅ `lab_name` - 实验室名称

### 前端问题

**修改前的列定义**：

```javascript
{
  title: '设备',
  dataIndex: 'equip_id',    // ❌ 错误：显示的是设备ID
  key: 'equip_id',
},
{
  title: '实验室',
  dataIndex: 'lab_id',      // ❌ 错误：显示的是实验室ID（字段不存在）
  key: 'lab_id',
},
```

**问题**：
1. 设备列绑定到 `equip_id`，显示的是ID而不是名称
2. 实验室列绑定到 `lab_id`，但后端返回的是 `lab_code` 和 `lab_name`

## 📊 数据格式说明

### 注意事项

在不同的接口中，`equip_name` 的格式可能不同：

1. **我的借记页面**：
   ```javascript
   equip_name: "显微镜"  // ✅ 字符串格式
   ```

2. **审批管理/所有借记页面**：
   ```javascript
   equip_name: ["显微镜"]  // ✅ 数组格式
   ```

**处理方式**：

```javascript
// 字符串格式（我的借记）
render: (text) => text || '-'

// 数组格式（审批管理/所有借记）
render: (text) => text?.[0] || '-'
```
