# LabEquip-BMS 新手启动指南

面向第一次接触本项目的开发者：说明项目由哪些部分组成、需要哪些中间件，以及如何在本机把前后端跑起来。

**目录**：一、项目组成 → 二、需要哪些中间件 → **三、安装开发环境（Node / npm / Go / MySQL）** → 四、方式 A 本机运行 → 五、方式 B Docker → 六、常见问题 → 七、相关文件

---

## 一、项目组成

| 部分 | 技术 | 作用 |
|------|------|------|
| 前端 | React（`react-scripts`） | 浏览器中的管理界面 |
| 后端 | Go（Gin 等） | 提供 `/api` 等业务接口 |
| 数据 | MySQL | 用户、实验室、器材、借还记录等 |

**Nginx**：本地开发**不是必须**；主要用于生产环境做反向代理、静态资源与域名。开发阶段用前端自带的开发服务器即可。

---

## 二、需要哪些中间件？

### 本地开发「跑通」所必需

| 软件 | 说明 |
|------|------|
| **MySQL** 5.7+ / 8.0 | 必须。初始化脚本会创建 `bms_basic_db`、`bms_inv_db`、`bms_task_db`，与 `server/_config/conf.yaml` 中的数据源对应。 |
| **Go** 1.17+（建议 1.21+） | 必须。用于编译、运行后端。 |
| **Node.js** + **npm** | 必须。建议 **Node 18 或 20 LTS**；若使用 Node 17+，需使用下文「六、常见问题」中的 OpenSSL 处理方式。 |

### 配置文件里出现的其他组件

`server/_config/conf.yaml` 中可能包含 Redis、Kafka、Elasticsearch 等配置（多从通用模板继承）。**常规启动流程主要依赖 MySQL**。若某功能在运行时明确报错连接 Redis/Kafka 等，再按需部署或调整配置。

### 可选

| 软件 | 说明 |
|------|------|
| **Nginx** | 生产或模拟生产时使用；纯本地开发可不装。 |
| **Docker** | 可用 `docker-compose` 启动 MySQL + 后端等，见下文「方式 B」。 |

## 三、安装开发环境（Node、npm、Go、MySQL）

下面按「先搞懂 → 再安装 → 最后验证」来写；**前端只依赖 Node.js 与 npm**，后端依赖 Go，全项目依赖 MySQL。

### 1. Node.js 和 npm 是什么关系？

- **Node.js**：让 JavaScript 能在电脑本机运行的环境；本项目的 React 前端用 Node 来执行 `npm install`、开发服务器、打包等。
- **npm**（Node Package Manager）：**随 Node.js 一起提供**的包管理工具，用来下载 `frontend/package.json` 里列出的依赖（如 React、antd）。装好 Node 后，终端里一般就有 `npm` 命令。

验证是否装好（版本号有输出即可，具体数字不必完全一致）：

```bash
node -v
npm -v
```

### 2. Node 装哪个版本？

- **推荐**：**Node 18 LTS** 或 **Node 20 LTS**（长期支持版，兼容本项目的前端工具链最省心）。
- **不推荐**：Node 16 已结束官方支持；Node 22 也能用，但若遇到 OpenSSL/Webpack 报错，见下文「五、常见问题」或改用 Node 20 LTS。

### 3. macOS 上安装 Node + npm

**方式 A：官网安装包（适合小白）**

1. 打开：https://nodejs.org/zh-cn  
2. 下载 **LTS** 版本（例如 20.x）的 `.pkg`，双击按提示安装。  
3. 重新打开终端，执行 `node -v`、`npm -v` 检查。

**方式 B：Homebrew（已安装 [Homebrew](https://brew.sh/) 时）**

```bash
brew install node@20
```

若 `node` 命令找不到，按 `brew` 提示把 `node@20` 加入 PATH（终端里会打印 `echo` / `export` 一类说明）。

**方式 C：nvm（需要多版本切换时）**

```bash
# 安装 nvm（以官方文档为准：https://github.com/nvm-sh/nvm ）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
# 重新打开终端后：
nvm install 20
nvm use 20
node -v
npm -v
```

### 4. Windows 上安装 Node + npm

**方式 A：官网安装包（适合小白）**

1. 打开：https://nodejs.org/zh-cn  
2. 下载 **LTS** 的 `.msi`，安装时保持默认选项（会勾选把 Node 加入 PATH）。  
3. **关闭并重新打开**「命令提示符」或 PowerShell，执行：

```cmd
node -v
npm -v
```

**方式 B：[nvm-windows](https://github.com/coreybutler/nvm-windows)（需要多版本时）**

安装 nvm-windows 后，例如：

```cmd
nvm install 20.18.0
nvm use 20.18.0
node -v
npm -v
```

> **说明**：本仓库 `frontend/package.json` 里 `start` 脚本使用 `NODE_OPTIONS=...` 这种写法，是 **Linux/macOS** 常见写法。若在 **Windows CMD** 下报错，可改用 **Git Bash**、**PowerShell** 里设置环境变量，或安装依赖后手动设置：  
> `set NODE_OPTIONS=--openssl-legacy-provider`（CMD）再执行 `npm start`；更省事的做法是改用 **Node 20 LTS**，多数情况下无需该参数。

### 5. Linux（以 Ubuntu / Debian 为例）安装 Node + npm

**方式 A：用 NodeSource 安装 LTS（版本较新，推荐）**

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v
npm -v
```

**方式 B：用 nvm（用户目录安装，无需 root）**

与 macOS 类似，安装 nvm 后执行 `nvm install 20`、`nvm use 20`。

> 仅用系统自带的 `apt install npm` 有时版本很旧，可能导致与本项目前端依赖不兼容，**优先用 LTS 的 Node 20**。

### 6. npm 下载慢？（可选）配置国内镜像

仅影响「从网络拉包」的速度，**不是必须**：

```bash
npm config set registry https://registry.npmmirror.com
npm config get registry   # 确认已生效
```

恢复官方源（需要时）：

```bash
npm config delete registry
```

### 7. 安装 Go（后端）

- 官网：https://go.dev/dl/  
- macOS：下载 `.pkg` 安装，或使用 `brew install go`。  
- Windows：下载 `.msi` 安装，安装后新开终端执行 `go version`。  
- Linux：可用包管理器或下载压缩包，并配置 `PATH`（详见 Go 官方文档）。

本项目要求 **Go 1.17+**，建议 **1.21+**。

### 8. 安装 MySQL

- **macOS**：`brew install mysql@8.0`，`brew services start mysql@8.0`（详见项目根目录 `README.md` 或 `DEPLOY.md`）。  
- **Windows**：从 MySQL 官网下载安装程序，记住 root 密码。  
- **Linux**：`sudo apt install mysql-server`（或发行版对应命令），启动服务并设置 root 密码。

安装完成后，确保能用客户端或命令行登录，并能执行项目根目录下的 `database_schema.sql`。

### 9. 环境一览检查清单

在继续「方式 A」之前，建议在终端确认：

| 命令 | 期望 |
|------|------|
| `node -v` | v18.x 或 v20.x（推荐） |
| `npm -v` | 有版本号即可 |
| `go version` | 显示 go1.17 及以上 |
| `mysql --version` 或能连上 MySQL | 能执行导入 SQL |

## 四、方式 A：本机直接运行（推荐新手）

### 1. 确认环境已就绪

完成上文 **「三、安装开发环境」**，保证 Node、npm、Go、MySQL 可用。

### 2. 初始化数据库

在**项目根目录**执行（将 `-p` 后的密码交互换成你的 MySQL root 密码）：

```bash
mysql -u root -p < database_schema.sql
```

脚本会创建三个数据库及表结构。

### 3. 修改数据库连接（如需要）

编辑 `server/_config/conf.yaml`，其中默认类似：

```text
root:123456@(127.0.0.1:3306)/bms_basic_db
```

若你的 MySQL 账号密码不是 `root` / `123456`，请同步修改所有 `masterDsn`、`replicasDsn` 中的账号、密码与地址，保持三套数据源（`default`、`inv`、`bms`）一致。

### 4. 启动后端

**必须在项目根目录 `LabEquip-BMS` 下执行**（后端用当前工作目录定位 `server/_config`）：

```bash
cd /path/to/LabEquip-BMS
go mod download
go run ./server/api/main.go
```

日志中出现 API 监听地址（如 `0.0.0.0:8080`）即表示启动成功。

> **注意**：不要使用 `cd server/api && go run main.go`，否则工作目录不对，配置文件路径会错误。

### 5. 启动前端

新开一个终端：

```bash
cd /path/to/LabEquip-BMS/frontend
npm install
npm start
```

`frontend/package.json` 中配置了 `proxy` 指向 `http://localhost:8080`，开发时前端会把 API 请求转发到后端。

### 6. 访问系统

浏览器打开：**http://localhost:3000**

## 五、方式 B：Docker Compose（了解即可）

仓库提供 `docker-compose.yml` 与 `deploy.sh`，可启动 MySQL、后端、Nginx 等容器。

使用前请阅读 **`DEPLOY.md`**，并注意：

1. **数据库地址**：在容器网络中，应将 `conf.yaml` 里的主机从 `127.0.0.1` 改为 **`mysql`**（与 compose 中的服务名一致），密码与 compose 中环境变量一致。
2. **前端静态资源**：若 Nginx 挂载本地目录 `frontend/build`，需先在宿主机执行 `npm run build` 生成静态文件，否则页面可能为空。
3. Nginx 示例配置可能针对本机开发（如代理到 `127.0.0.1:3000`），与纯容器部署不完全相同；若出现 502 或空白页，需对照 `DEPLOY.md` 与实际挂载的配置调整。

**建议**：先按「方式 A」在本机跑通，再尝试 Docker。

## 六、常见问题

### 1. `Error: error:0308010C:digital envelope routines::unsupported`

原因：Node 17+ 与旧版 Webpack（`react-scripts` 4.x）在 OpenSSL 3 下的兼容问题。

处理：本项目已在 `package.json` 的 `start` / `build` / `test` 中加入 `NODE_OPTIONS=--openssl-legacy-provider`。若仍异常，可改用 **Node 20 LTS**。

### 2. 后端启动失败或找不到配置

确认是否在**项目根目录**执行 `go run ./server/api/main.go`。

### 3. 前端能访问但接口失败

- 确认后端已启动且监听 `8080`。
- 确认 MySQL 已启动，且 `conf.yaml` 中账号密码与库已导入。
- 浏览器开发者工具 Network 中查看 `/api` 请求状态码与返回内容。

### 4. 与 README「快速开始」的差异

若 README 中写 `cd server/api` 再 `go run main.go`，请以本文为准：**在仓库根目录执行** `go run ./server/api/main.go`。

### 5. `node` / `npm` 提示「找不到命令」或不是内部命令

- **macOS / Linux**：确认已安装 Node，并**重新打开终端**；若用 nvm，确认已执行 `nvm use` 且 nvm 初始化脚本已写入 `~/.zshrc` 或 `~/.bashrc`。  
- **Windows**：重新打开终端；在「系统环境变量」中检查是否包含 Node 的安装目录；安装时勾选「Add to PATH」。

### 6. `npm install` 很慢或超时

可先配置国内镜像（见「三、6. npm 下载慢」），或检查网络/代理后再执行 `npm install`。
