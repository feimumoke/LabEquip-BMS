# 三级账字段说明

## 📊 新增显示字段

根据用户需求，三级账（Transaction Log）页面现在显示以下所有数量字段：

| 字段名 | 英文字段 | 说明 | 颜色标识 |
|--------|---------|------|---------|
| **操作数量** | `op_qty` | 本次操作的数量（增加为正，减少为负） | 绿色(+)/红色(-) |
| **总数** | `total_qty` | 总库存数量 | 加粗黑色 |
| **在手** | `on_hand_qty` | 在实验室库存 = total - borrowed | 黑色 |
| **可用** | `avail_qty` | 可用库存 = total - borrowed - allocated | 蓝色 |
| **借出** | `borrowed_qty` | 当前被用户实际借走的（已经拿走了） | 红色 |
| **分配** | `allocated_qty` | 已经分配但未拿走的 | 橙色 |

---

## 📐 数量关系说明

### 基本关系

```
总数 (total_qty)
  ├─ 在手 (on_hand_qty) = total - borrowed
  └─ 可用 (avail_qty) = total - borrowed - allocated
      ├─ 借出 (borrowed_qty) - 已拿走，影响在手数量
      └─ 分配 (allocated_qty) - 已分配未拿走，影响可用数量
```

### 详细说明

#### 1. **总数 (total_qty)**
- **定义**：该设备在该实验室的总库存数量
- **变化时机**：
  - 增加库存操作 → 总数增加
  - 扣减库存操作 → 总数减少

#### 2. **在手 (on_hand_qty)**
- **定义**：在实验室的实际库存数量
- **计算公式**：`total_qty - borrowed_qty`
- **含义**：设备在实验室，但可能已分配给学生（尚未拿走）

#### 3. **可用 (avail_qty)**
- **定义**：可以被分配的库存数量
- **计算公式**：`total_qty - borrowed_qty - allocated_qty`
- **含义**：既没有被借走，也没有被分配

#### 4. **借出 (borrowed_qty)**
- **定义**：学生已经拿走的数量
- **变化时机**：
  - 学生拿走设备 → 借出数量增加
  - 学生归还设备 → 借出数量减少
- **影响**：减少"在手"和"可用"数量

#### 5. **分配 (allocated_qty)**
- **定义**：已分配但学生尚未拿走的数量
- **变化时机**：
  - 审批通过 → 分配数量增加
  - 学生拿走 → 分配数量减少，借出数量增加
  - 取消/拒绝 → 分配数量减少
- **影响**：只减少"可用"数量，不影响"在手"数量

## 📝 后端实现

### 1. Proto 定义

**文件**：`api_idl/apps/bms/search_inv_transaction.proto`

```protobuf
message InvTransaction {
  required string transaction_id = 1;
  required string sheet_id = 2;
  required string equip_id = 3;
  repeated string equip_name = 4;
  optional string lab_code = 5;
  optional string lab_name = 6;
  optional string operator = 7;
  optional string remark = 8;
  optional int64 total_qty = 14;        // 总数
  optional int64 on_hand_qty = 15;      // 在手数量
  optional int64 avail_qty = 13;        // 可用库存
  optional int64 borrowed_qty = 16;     // 借出数量
  optional int64 allocated_qty = 17;    // 分配数量
  optional int64 op_qty = 9;            // 操作数量
  optional int64 trans_type = 10;
  optional int64 sheet_type = 11;
  optional int64 ctime = 12;
}
```

### 3. 数据库表

**表名**：`transaction_log_tab`

```sql
CREATE TABLE `transaction_log_tab` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `transaction_id` varchar(64) NOT NULL COMMENT '交易ID',
  `sheet_id` varchar(64) NOT NULL COMMENT '单据ID',
  `equip_id` varchar(64) NOT NULL COMMENT '设备ID',
  `lab_id` varchar(64) NOT NULL COMMENT '实验室ID',
  `operator` varchar(128) NOT NULL COMMENT '操作人',
  `remark` varchar(512) DEFAULT NULL COMMENT '备注',
  `total_qty` bigint NOT NULL COMMENT '总数',
  `on_hand_qty` bigint NOT NULL COMMENT '在手数量',
  `available_qty` bigint NOT NULL COMMENT '可用库存',
  `borrowed_qty` bigint NOT NULL COMMENT '借出数量',
  `allocated_qty` bigint NOT NULL COMMENT '分配数量',
  `op_qty` bigint NOT NULL COMMENT '操作数量',
  `trans_type` tinyint NOT NULL COMMENT '交易类型',
  `sheet_type` tinyint NOT NULL COMMENT '单据类型',
  `ctime` bigint NOT NULL COMMENT '时间戳',
  PRIMARY KEY (`id`),
  KEY `idx_transaction_id` (`transaction_id`),
  KEY `idx_sheet_id` (`sheet_id`),
  KEY `idx_equip_lab` (`equip_id`, `lab_id`),
  KEY `idx_ctime` (`ctime`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 💡 使用建议

1. **查看库存变化**：通过"操作数量"和"总数"列，快速了解库存增减情况
2. **监控借出状态**：通过"借出"和"分配"列，了解设备借出情况
3. **库存可用性**：通过"可用"列，快速判断是否有设备可以借出
4. **审计追溯**：通过完整的数量记录，可以追溯任何时间点的库存状态
